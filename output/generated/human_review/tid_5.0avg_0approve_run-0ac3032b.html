2. Technical Invention Disclosure

2.1 Problem Statement

A TDX guest must sometimes receive packets into host-visible shared pages so the device or hypervisor-backed virtio path can DMA them, but Linux generic receive processing assumes an skb may later be tapped, cloned, queued, linearized, decrypted, segmented, reassembled, steered across CPUs, or retained on socket queues for an arbitrarily long time. In this disclosure, "shared" means guest pages intentionally mapped with the TDX shared attribute so host/device agents may access them; "private" means TDX-private guest memory not host-readable. "Plaintext exposure" means any received byte retained in shared memory after the queue-local RX lease should have ended, including payload bytes and any inner payload produced after tunnel, TLS, or IPsec decryption/decapsulation. Outer L2/L3/L4 headers may be read while shared only for classification and GRO eligibility, but no write into shared-backed bytes is allowed. Existing ingress fanout makes the hazard concrete: GRO keeps packet state across multiple arrivals; netif_receive_skb_list_internal() and netif_receive_skb_internal() feed __netif_receive_skb_core(); __netif_receive_skb_core() can reach AF_PACKET/ptype_all taps, tc/clsact ingress, netfilter ingress and prerouting, xfrm/IPsec, tunnel decapsulation, and protocol stacks; enqueue_to_backlog() enables RPS/RFS cross-CPU retention; and later helpers such as skb_clone(), skb_segment(), pskb_expand_head(), skb_cow(), and socket queueing can extend lifetime further. Separately, TX notification overhead is amplified in TDX because guest-to-host notifications trigger expensive exits. Existing virtio use of EVENT_IDX and generic interrupt moderation reduces some host-visible interrupts but does not define when the guest should defer or emit its own notify attempts, and it does not provide a bounded-first-kick rule after a queue transitions from idle to busy.

2.2 Prior-Art Gap

Prior art clusters into four insufficient categories. First, private-first or bounce-buffer RX copies every packet before the stack knows whether the packet will be dropped, coalesced, or remain queue-local, so it misses the confidentiality-preserving but copy-minimizing lease architecture. Second, page provenance tagging by itself is not enough because it does not identify a single unskippable Linux boundary before __netif_receive_skb_core()-reachable consumers and does not recurse through skb frags, frag_list, clones, and segmentation products. Third, generic interrupt moderation and VIRTIO_RING_F_EVENT_IDX operate on interrupt suppression or device-side event visibility, not on a guest-side state machine for deferring notify attempts with precise IDLE->BUSY, BUSY->IDLE, and forced-flush transitions. Fourth, prior batching schemes typically say only "batch then kick later"; they do not claim the invariant that the first deferred notify attempt after idle-to-busy must be emitted by earliest-of elapsed time T, accumulated descriptors N, accumulated bytes M, or sustained-backlog rate cap R, while preserving virtqueue_kick_prepare() semantics under the queue lock. The disclosed combination is therefore not a generic tag-and-copy system and not ordinary virtio coalescing; it is a TDX-specific contract between guest memory visibility, Linux ingress lifetime boundaries, and bounded exit-aware virtqueue notification.

2.3 Proposed Invention

The invention has two independent modules and one combined embodiment. Module A is a Mandatory Promotion Gate (MPG) for TDX shared RX memory. Each receive queue owns a lease generation and allocates shared page_pool pages marked in queue-local metadata as shared-lease pages. Any skb that references such bytes carries a shared-byte tag plus a compact descriptor of which linear region, frags, and frag_list elements are shared-backed. GRO merge state inherits the tag if any contributor is shared-backed. The mandatory gate is placed at the entry of netif_receive_skb_list_internal() and netif_receive_skb_internal(), after GRO/list consolidation but before any path into __netif_receive_skb_core(); enqueue_to_backlog() is additionally fenced so cross-CPU handoff cannot occur first. The gate runs tdx_rx_promote_skb(), a recursive promotion routine that allocates private storage, copies every shared-backed byte in the linear head and frags, reconstructs frag_list members privately, copies skb metadata, and clears the tag only when no shared-backed byte remains reachable. Defensive assertions reject or warn if a tagged skb reaches clone, segmentation, COW, or queueing helpers outside the sanctioned lease domain. Module B is a Notification Epoch Controller (NEC) for virtio TX. An "attempt" event occurs exactly when the driver would normally call virtqueue_kick_prepare() after publishing descriptors. NEC state per send queue comprises IDLE or BUSY phase, current epoch generation, first_attempt_time, deferred descriptor and byte counters, last_notify_time, pending_notify bit, and live bit. On IDLE->BUSY, the first attempt may be deferred, but NEC arms a pinned hrtimer deadline and records the first deferred attempt. BUSY continues accumulating attempts. The epoch is force-closed at earliest-of: elapsed time T from first deferred attempt, accumulated deferred descriptors >= N, accumulated deferred bytes >= M, or elapsed time since last notify >= rate cap R under sustained backlog. A flush path under the send-queue lock calls virtqueue_kick_prepare(); if prepare returns true, the queue notifies, otherwise pending state is cleared because EVENT_IDX already suppresses the need. BUSY->IDLE occurs only when no unpublished notification remains pending and the queue is drained. The combined embodiment uses MPG on RX and NEC on TX in the same TDX guest networking stack.

2.4 Architecture

Control-flow and invariants:

RX path
shared DMA page_pool page (lease_gen=q.gen, shared=1)
    -> driver/NAPI builds skb or GRO state, tag = SHARED_RX if any byte references shared page
    -> optional read-only parse for GRO/classification eligibility
    -> if XDP_PASS, continue; if XDP_REDIRECT/XDP_TX/AF_XDP zero-copy enabled, queue falls back to private-first mode
    -> napi_gro_receive()/napi_gro_frags() may aggregate, tag propagates if any contributor is shared
    -> gro_normal_list / skb list finalized
    -> netif_receive_skb_list_internal() or netif_receive_skb_internal()
         -> if skb_has_shared_rx_bytes() => tdx_rx_promote_skb_list() / tdx_rx_promote_skb()
         -> recursive inspection of linear head, nr_frags, frag_list
         -> allocate private skb/pages, copy shared-backed bytes, preserve metadata, clear SHARED_RX only on full replacement
    -> ASSERT: no shared-backed skb may enter __netif_receive_skb_core()
    -> __netif_receive_skb_core() may now reach ptype_all, tc ingress, netfilter, xfrm/IPsec, tunnels, protocols
    -> enqueue_to_backlog() contains identical fence so RPS/RFS cannot queue a tagged skb cross-CPU

TX path
sendmsg/xmit publishes descriptors under per-send-queue lock
    -> ATTEMPT event: point where driver would call virtqueue_kick_prepare()
    -> NEC state machine
       IDLE --first attempt--> BUSY(epoch opened, counters reset, deadline armed)
       BUSY --attempts accumulate--> BUSY
       BUSY --earliest of T/N/M/R--> FLUSH
       FLUSH under queue lock: virtqueue_kick_prepare(); if true virtqueue_notify(); update last_notify_time; clear pending
       BUSY --queue drained && !pending--> IDLE
    -> hrtimer callback never grabs queue lock; it only queues local irq_work
    -> irq_work reacquires queue lock and performs FLUSH if epoch/live bits still match

Data representation:
- shared/private status is stored in page_pool lease metadata and skb shared-byte tags, not a new ABI-visible skb layout
- lease_gen increments on reset/suspend/netdev-down so old shared pages cannot silently re-enter a new lease
- a page is recyclable only if page_pool ownership is restored, shared_refcnt is zero, and page.lease_gen matches the currently valid generation or the pool is being destroyed

Ingress scope:
- In scope: NAPI, GRO variants, busy-poll paths that end in netif_receive_skb_list_internal()/netif_receive_skb_internal(), packet taps, tc/clsact, netfilter, xfrm, protocol receive paths, RPS/RFS backlog
- Out of scope by explicit exclusion or fallback: AF_XDP zero-copy, XDP_REDIRECT, XDP_TX, and netpoll on queues configured for shared leases; enabling them forces private-first RX on those queues

2.5 Implementation Plan

1. Add queue-local RX lease metadata beside the receive queue and page_pool allocation context: current lease_gen, shared_refcnt accounting, and a compact per-page descriptor indicating that the page was provisioned as TDX-shared. 2. Add skb-side shared-byte provenance: a SHARED_RX tag plus recursive bookkeeping for linear head, frags, and frag_list. Tag propagation rules are explicit: GRO output is tagged if any input was tagged; skb_clone() inherits the tag; skb_segment() copies the tag to all children until promotion; any helper that would write into tagged bytes must either force promotion first or reject that path. 3. Place the mandatory promotion helper at the entry to netif_receive_skb_list_internal() and netif_receive_skb_internal(). The helper walks the skb recursively, copies only shared-backed regions into new private storage, reconstructs frag_list chains privately, preserves checksum/hash/mark/timestamp metadata, and clears the tag only after the final recursive pass proves no shared-backed byte remains reachable. 4. Fence cross-CPU retention by adding the same predicate to enqueue_to_backlog(), guaranteeing that RPS/RFS cannot enqueue shared-backed skbs before promotion. 5. Enforce the no-write-on-shared rule: while SHARED_RX is set, header parsing may read bytes already linearized by the driver, but helpers that may mutate or pull data such as pskb_may_pull(), skb_store_bits(), skb_cow(), pskb_expand_head(), software checksum writeback, or linearization cause immediate promotion. 6. Integrate mode exclusions: shared-lease queues reject AF_XDP zero-copy registration, XDP_REDIRECT, and XDP_TX, and they fall back to private-first RX if netpoll is enabled. 7. Implement NEC state in the virtio send queue: phase, epoch_gen, first_attempt_ns, last_notify_ns, deferred_descs, deferred_bytes, pending_notify, timer_armed, and live. The authoritative lock is the existing per-send-queue lock already used for descriptor publication. The lock order is queue lock first and only; no NEC path takes a second virtqueue-side lock while holding another unrelated lock. 8. In the xmit path, replace direct kick calls with nec_attempt(descs, bytes). If thresholds are not yet met, the epoch remains BUSY and the hrtimer is armed only on the first IDLE->BUSY transition. 9. The hrtimer callback is lockless and queues local irq_work; irq_work reacquires the send-queue lock, revalidates epoch_gen/live, calls virtqueue_kick_prepare(), notifies only if prepare returns true, and clears pending state. 10. Teardown protocol: under netdev down, reset, suspend, or queue teardown, mark live=0, increment epoch_gen and lease_gen, cancel the hrtimer, synchronize or flush irq_work, then detach the virtqueue/page_pool. This ordering prevents callbacks from touching a dead vq and prevents old shared pages from re-entering a new lease generation.

2.6 Validation Plan

Validation has four layers. First, ingress invariants: instrument debug assertions that WARN_ON_ONCE if skb_has_shared_rx_bytes() is observed at __netif_receive_skb_core(), packet taps, tc ingress entry, netfilter ingress/prerouting hooks, xfrm receive paths, enqueue_to_backlog(), socket queueing, skb_clone(), or skb_segment() without prior promotion. Second, confidentiality semantics: test mixed layouts with linear head plus page frags plus frag_list, including GRO merge/flush, tunnel decapsulation, IPsec or TLS receive/decrypt, and packets that require pskb_may_pull() or software checksum fixup, verifying that such paths either promote first or fall back early and never retain host-visible plaintext. Third, lifecycle proof: exercise reset, suspend/resume, netdev down/up, NAPI disable/enable, and page_pool recycle while clones or GRO state still exist, and verify that old lease generations are quarantined until references drop to zero and no shared byte remains reachable. Fourth, NEC correctness and performance: run single-flow latency, many-flow throughput, and bursty microbenchmarks under virtio multiqueue with EVENT_IDX on and off, verifying the bounded-first-kick invariant, no lost notifications, no use-after-free during teardown, and reduced guest exits per transmitted byte. The pass condition is both functional and auditable: every skb reaching __netif_receive_skb_core() is private-only, and every deferred first kick after IDLE->BUSY is emitted by earliest-of T, N, M, or R.

4. Risks & Mitigations

  • Risk: Copy cost at promotion may erase performance gains on small packets. Mitigation: keep the lease queue-local through GRO/list consolidation so multiple shared arrivals can collapse into one promotion, and allow driver fallback to private-first RX for traffic mixes dominated by tiny packets or write-heavy parsing.
  • Risk: Some ingress paths bypass the intended boundary. Mitigation: define scope precisely: any path entering generic skb receive must pass through netif_receive_skb_list_internal() or netif_receive_skb_internal(); queues configured for shared leases explicitly disable or fall back on AF_XDP zero-copy, XDP_REDIRECT, XDP_TX, and netpoll.
  • Risk: Header helpers may accidentally write into shared bytes before promotion. Mitigation: only permit read-only parsing while shared; force immediate promotion before pskb_may_pull(), checksum writeback, skb_cow(), linearization, or any helper that can mutate packet data.
  • Risk: NEC could race with virtqueue teardown or lose a notify. Mitigation: use one authoritative send-queue lock, a lockless hrtimer callback that only queues local irq_work, epoch generation numbers, and strict teardown ordering of live=0 -> cancel timer -> sync irq_work -> detach vq.
  • Risk: Prior-art challenge that NEC is just standard batching. Mitigation: claim the precise attempt definition, explicit state transitions, and the bounded-first-kick earliest-of(T,N,M,R) invariant after IDLE->BUSY, which generic batching and EVENT_IDX suppression do not specify.
  • Risk: Page_pool recycling may reissue old shared pages across reset/suspend. Mitigation: increment lease_gen on lifecycle transitions, quarantine pages from retired generations until their reference counts drop to zero, and require generation match plus zero shared_refcnt before reuse.
