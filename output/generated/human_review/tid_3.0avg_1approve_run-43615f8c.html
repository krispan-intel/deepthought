1. Executive Summary

One-Liner

Compile an ordered cls_flower chain into an immutable RCU-published epoch of stable rule slots and byte-position-major value/mask windows; when ingress classification is already running on a threaded-NAPI poll kthread, extract a bounded canonical flower key for a small packet microbatch, generate an exact conservative candidate bitmap by evaluating masked-equality necessary conditions across many masks at once using integer SIMD and optionally x86 AMX tile reduction, then invoke the unchanged scalar cls_flower priority walk and tc actions only for surviving candidates.
Novelty: The inventive step is not "use AMX/SIMD for networking". It is a semantics-preserving bridge that turns an ordered heterogeneous cls_flower chain into immutable epoch data where selected canonical key bytes are laid out
Feasibility: The design is implementable because it reuses cls_flower's existing field semantics and confirmation path, pushes heavy epoch compilation entirely off the fast path under RTNL, pub

2. Technical Invention Disclosure

2.1 Problem Statement

cls_flower software classification becomes expensive when many priority-ordered rules carry diverse masks across L2, L3, L4, tunnel, conntrack, and skb metadata fields. Existing mask-sharing and hash-based lookups help when masks cluster, but they still require per-mask lookup paths and scalar confirmation under original priority semantics. The bottleneck is especially visible in software ingress paths where operators need exact tc behavior but cannot assume NIC offload or reauthor policy in another framework.
