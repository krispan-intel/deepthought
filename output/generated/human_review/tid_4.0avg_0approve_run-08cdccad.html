2. Technical Invention Disclosure

2.1 Problem Statement

Deep package, die, or LLC-cluster idle residency is often blocked not by runnable work but by interrupt delivery landing on CPUs inside an otherwise quiescent island. Linux has cpuidle, nohz, topology, irq affinity, MSI management, interrupt remapping, and timer broadcast machinery, but it lacks a topology-keyed runtime state machine that: (a) defines a power island concretely in Linux-visible terms; (b) distinguishes delivery-limited idle loss from activity-limited idle loss using kernel-observable evidence; (c) evacuates only safe managed MSI/MSI-X interrupts to effective destinations outside the island under concurrent affinity writers, hotplug, device removal, and vector migration; and (d) gives cpuidle a formally correct committed-next-expiry bound that remains valid across local-timer versus broadcast-timer ownership changes and clockevent replacement.
