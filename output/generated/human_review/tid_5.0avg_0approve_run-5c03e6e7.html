2. Technical Invention Disclosure

2.1 Problem Statement

Intel RAPL exposes a small, quantized actuator space per domain rather than a continuous power-control plane. For a given package, die, or DRAM domain, the writable controls are typically a subset of PL1, PL2, clamp, and an encoded averaging window, subject to presence bits, lock bits, unit scaling, and platform-specific min and max bounds. Linux already has multiple subsystem-level signals that imply different preferred envelopes, including scheduler urgency, PM QoS latency pressure, memory PSI, reclaim intensity, and microarchitectural phase signals observable through Intel PMU counters. Existing practice is fragmented: static caps ignore phase changes, userspace daemons react slowly and race with firmware or sysfs writers, and generic governors reason in abstract watts or errors without treating the encoded RAPL window as a direct actuator dimension. The result is oscillation, undefined coexistence, poor hotplug behavior, and ambiguous failure handling. A kernel-resident policy local to intel_rapl is needed to fuse subsystem intents and PMU-qualified telemetry into deterministic, bounded, package-scope decisions that respect lock bits, coexist with other writers, avoid atomic-context work, and remain stable across Xeon generations.

2.5 Implementation Plan

Implementation proceeds in five increments. First, add the local lease store, snapshot compiler, and encoded-unit tuple representation to the intel_rapl driver without enabling PMU input. Second, add domain-specific capability discovery, baseline tuple capture, and the serialized commit worker with readback verification and guard-tuple semantics. Third, add writer-coexistence detection and inhibit-shadow state machines using normalized encoded readback. Fourth, add owner-CPU perf sampling with per-CPU sample slots, package coverage accounting, and PMU-quality-gated advisory lease emission. Fifth, populate a model table for event selection and platform policy mapping on selected Xeon families, while keeping the PMU channel disabled on unsupported models rather than guessing. At no step does the design require a new userspace ABI or changes to generic mm or scheduler hot paths.

2.6 Validation Plan

Validate correctness and stability on at least one recent Xeon server platform with package and, where available, die or DRAM domains. Compare against stock intel_rapl plus userspace powercap control under bursty web, mixed latency-sensitive, and Linpack-like compute phases. Measure tuple oscillation rate, time spent in inhibit or shadow mode, PMU quality-pass rate, powercap write frequency, readback mismatch frequency, and workload throughput-per-joule. Inject CPU hotplug, perf multiplexing, counter resets, sysfs writer races, and artificial out-of-band MSR changes to confirm deterministic backoff. Verify that no advisory lease is emitted when enabled-running coverage falls below threshold and that fallback reversion never exceeds hardware or administrator bounds.

4. Risks & Mitigations

  • Risk: PMU event portability varies across Xeon generations. Mitigation: use a CPU-model table, require explicit event support, and disable the PMU advisory channel entirely on unqualified models.
  • Risk: frequent userspace or firmware writes can starve the local governor. Mitigation: deterministic INHIBIT and SHADOW modes make the policy observe-only rather than oscillatory, while still maintaining shadow targets for later re-entry.
  • Risk: hotplug or perf teardown could race policy compilation. Mitigation: compile from per-CPU sample buffers only, never from live perf_event pointers, and sequence offline through cancel_delayed_work_sync before event release.
  • Risk: encoded-window tuning could still oscillate on marginal workloads. Mitigation: encoded-space vote counters, one-step-per-interval movement, and asymmetric tighten-versus-relax thresholds bound churn.
  • Risk: overlapping with prior art could be alleged as generic governor logic. Mitigation: claim language and specification are narrowed to encoded-window-first compilation, PMU-quality-gated advisory leases, and deterministic readback-based writer coexistence.
