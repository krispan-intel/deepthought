2. Technical Invention Disclosure

2.1 Problem Statement

Linux today makes readahead sizing, folio-order selection, and writeback clustering decisions primarily from access-pattern history, refault distance, and generic locality heuristics. Those signals do not directly capture whether a particular file offset repeatedly causes expensive synchronous read misses, folio lock waits, folio writeback waits, or long-latency Intel memory accesses. Raw PEBS samples are not safe to translate into file offsets by walking VMAs or page tables in interrupt context, and using sampled address or sampled IP as a reverse-mapping source would create correctness, complexity, and prior-art overlap risk. At the same time, page-cache state is shared across tenants, so any feedback system that is not memcg-scoped can leak behavior or let one cgroup bias another cgroup's I/O pattern. The disclosed system solves these issues by using only strictly attributable kernel stall producers to define file-offset buckets, using PEBS only as gated reinforcement for already-attributed buckets, maintaining a separate per-mapping layout sequence for mutation safety, and limiting all actuation to bounded advisory clamps that never violate existing mm or writeback semantics.
