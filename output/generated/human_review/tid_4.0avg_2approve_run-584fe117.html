2. Technical Invention Disclosure

2.1 Problem Statement

Large x86 SMP systems pay avoidable synchronization cost in three places. In contended qspinlocks, multiple same-socket or same-die contenders repeatedly migrate the lock cacheline across packages because each contender independently enters the global queue. In scheduler remote wake paths, Linux already suppresses repeated nudges to the same destination CPU via first-transition pending semantics, but a source CPU waking many different destination CPUs in the same APIC logical cluster still commonly issues one architectural send per target or executes an iterative send_IPI_mask() loop. In Tree RCU grace-period forcing, the GP thread must act on authoritative qsmask state, yet today broad laggard nudging can still touch multiple related CPUs separately even when one topology-local steward could safely proxy the non-authoritative transport work. x86 APIC microarchitecture exposes topology hierarchy, logical clustering, and local deadline timers, but Linux cannot safely exploit those features by inventing new ownership rules, inferring quiescent states from delivery success, batching across semantically different nudge primitives, or trusting unstable firmware topology during hotplug, isolation, or virtualization churn.

2.2 Prior-Art Gap

Existing Linux behavior already covers important pieces in isolation: qspinlock already provides a compact queued lock; ttwu_queue_wakelist()/sched_ttwu_pending already suppress duplicate remote nudges per destination CPU with first-transition semantics; x86 APIC code already knows how to target masks; and Tree RCU already performs GP-driven forcing based on authoritative qsmask state. The gap is the absence of a single kernel-feasible bridge layer that: (1) converts CPUID/APIC topology into a conservative, epoch-versioned optimization-enablement map; (2) uses that map to bound acquisition-time representative gating ahead of qspinlock slowpath without changing stock unlock or PV/RT semantics; (3) batches only the final transport step for already-authoritative remote wake or RCU nudge events, splitting masks by primitive class and APIC cluster so behavior stays semantically identical to stock; and (4) proves immediate fail-closed reversion on collisions, hotplug drift, topology ambiguity, paravirt, PREEMPT_RT, isolation policy, or uncertain x2APIC transport cost. The claimed novelty is therefore not 'fewer wakeups' or 'generic APIC multicast,' but a correctness-preserving way to reduce architectural send operations and queue entries only when topology, primitive class, and policy all align.

2.3 Proposed Invention

Introduce a three-part fail-closed substrate. (1) Topology epoch manager: at boot and on CPU hotplug rebuild, parse CPUID leaf 0x1F when present, otherwise 0x0B, consuming only the x2APIC ID in EDX and the shift widths in EAX to derive stable package and cluster identifiers; package scope is always supported, while die scope is enabled only when 0x1F explicitly exposes a stable die level and Linux's published die IDs agree for all online CPUs. The result is RCU-published as a read-mostly topology snapshot with an incrementing epoch and a 'stable' bit. (2) qspinlock acquisition-time cohort gate: when a lock enters contended slowpath, a fixed boot-sized sidecar hash indexed by mixed lock address attempts to claim an entry with a lock key; if the bucket is occupied by another live key or the topology snapshot is unstable, the attempt goes straight to stock qspinlock slowpath. Otherwise, same-domain contenders increment a local-waiter counter, and at most one contender installs itself as representative by cmpxchg on rep_cpu plus a rep_gen token. Only that representative may enter stock queued_spin_lock_slowpath(); peers remain outside the global queue and spin on the sidecar using acquire loads, with explicit escape to stock if rep_gen does not advance within a bounded iteration or time budget, if rep_cpu becomes offline, if the topology epoch changes, or if batch_credit expires. Representative retirement uses release ordering to clear rep_cpu and advance rep_gen after unlock, so waiters can either elect a successor or enter drain-to-stock mode. (3) Subset-only APIC nudging: after scheduler or RCU code has already completed the stock authoritative per-target enqueue and pending-state transition, sender-local or GP-local code adds that target CPU to a transient bucket keyed by {topology epoch, APIC logical cluster, primitive class}. Primitive class is one of RESCHED_IPI, IRQ_WORK, or NONE/POLLING, and masks are never mixed across classes. On commit, the x86 hook may issue one logical- or cluster-mode x2APIC send for a bucket only if every target is within one legal cluster, every target requires the same primitive class, and the hardware path is no more expensive than the stock per-target loop; otherwise the exact stock helper is called per target. For Tree RCU specifically, only the GP thread may arm one housekeeping steward timer per lagging cohort and gp_seq, and the timer callback must re-read the current gp_seq and authoritative laggard mask under normal rcu_node locking before sending any nudge. A transient gp_seq-keyed bitmap suppresses duplicate non-authoritative nudges per target CPU within that GP and is reset on GP advance; it never contributes to quiescent-state accounting.

2.4 Architecture

Core objects:
1) `struct sync_topology_snapshot { u32 epoch; bool stable; enum boundary { PACKAGE, DIE }; cpumask_t online; cpumask_t housekeeping; u32 package_id[NR_CPUS]; u32 die_id[NR_CPUS]; u32 apic_cluster_id[NR_CPUS]; enum primitive_policy primitive_class[NR_CPUS]; }` published by RCU.
2) `struct qsc_entry { u64 key; u32 topo_epoch; u32 rep_gen; u16 rep_cpu; u16 rep_domain; u8 batch_credit; u8 flags; atomic_t local_waiters; }` in a fixed boot-time hash table.
3) `struct apic_nudge_batch { u32 topo_epoch; bucket[cluster_id][primitive_class] -> cpumask_t; }` as sender-local scratch for one lexical wake fan-out window only.
4) `struct rcu_cohort_nudge { u64 gp_seq; u32 topo_epoch; int steward_cpu; cpumask_t laggards; cpumask_t nudged_once; struct hrtimer timer; }` owned only by the current GP forcing path.

qspinlock state machine: `stock fastpath` -> on repeated failure and observed contention -> `lookup qsc_entry`; if invalid, `stock slowpath`; if valid and no representative for local domain, `elect representative`; representative alone enters stock `queued_spin_lock_slowpath`; peers wait on `rep_gen`; on representative unlock, `rep_gen++` and `rep_cpu=INVALID`; if any waiter exceeds `cohort_wait_iters` or `cohort_wait_ns`, or sees rep_cpu offline/topology epoch mismatch, it sets a drain-to-stock flag for that generation and bypasses the gate. Bounded unfairness means no domain may retain cohort preference past `MAX_LOCAL_PASSES` consecutive representative retirements or past the explicit age/iteration threshold; after either condition, all waiters for that sidecar generation bypass to stock until the bucket quiesces.

Scheduler correctness contract: for each destination CPU `d`, the authoritative operations remain stock: publish wakelist entry or remote runnable state, perform the same first-transition pending atomic used today, execute the same ordering edge (including the stock atomic/barrier pair such as `smp_mb__after_atomic()` where the current path requires it), then and only then add `d` to a sender-local scratch bucket. Commit may replace only the transport helper, never the publication or pending-state transition. If commit is skipped, every `d` is handled by the stock helper exactly once.

Primitive-class mapping: `RESCHED_IPI -> smp_send_reschedule()` semantics; `IRQ_WORK -> irq_work_queue_on()` semantics; `NONE/POLLING -> no architectural send, just stock polling-visible state`. CPUs whose current policy requires nohz_full/housekeeping-specific handling, polling-idle suppression, or any arch quirk that changes semantics are split into separate buckets or forced to singleton stock handling.

Tree RCU state machine: only `rcu_gp_kthread` inside the existing force-quiescent-state cadence may snapshot laggards from authoritative `qsmask` under the normal `rcu_node` lock chain, group them by topology cohort, and arm at most one steward timer per cohort when the same cohort remains lagging across a configurable number of FQS rounds. The steward is the lowest-numbered online housekeeping CPU in that cohort for the current topology epoch; if none exists, the cohort optimization is disabled. Timer expiry runs in non-sleeping context, rechecks `(current gp_seq == armed gp_seq)` with acquire ordering, re-snapshots the authoritative laggard subset under the normal lock, and uses `test_and_set_bit()` in `nudged_once` before issuing the stock-equivalent nudge transport. GP advance, CPU offline, topology epoch change, expedited GP, PREEMPT_RT, or rcu_nocb policy conflict cancels the timer and drops back to stock forcing.

2.5 Implementation Plan

1) Add an x86-only topology snapshot builder that consumes CPUID 0x1F first and 0x0B second, stores package and legal cluster IDs, records whether explicit die IDs are stable, and RCU-publishes snapshots with an incrementing epoch from CPUHP online/dying callbacks plus a sleeping rebuild worker; every consumer must compare epochs and abort on mismatch. 2) Add a boot-sized qspinlock sidecar table and a static key so the acquisition-time cohort gate activates only on x86, !PREEMPT_RT, !paravirt, stable topology, and non-colliding buckets; implement representative election with cmpxchg on `rep_cpu`, release/acquire ordering on `rep_gen`, explicit `MAX_LOCAL_PASSES`, `cohort_wait_iters`, and `cohort_wait_ns` escapes, and drain-to-stock mode when any waiter detects stalling or offline representative. 3) Add a sender-local scheduler batching helper whose lifetime is one wake fan-out loop or one wakelist drain only; keep current enqueue and pending-flag code unchanged, but collect post-publication targets into buckets by `{cluster, primitive_class}` and call a new `arch_apic_send_nudge_mask(class, mask)` hook only when the mask is intra-cluster and the hook reports a bounded hardware send count no worse than baseline. 4) Implement the x86 transport hook using only already-supported logical or cluster destination modes under x2APIC; if the mask spans clusters, the delivery mode is unavailable, the primitive class is unsupported, or the projected send count is not lower, return false so the caller executes the exact stock per-target helper sequence. 5) Add Tree RCU steward sidecars owned by the GP path: snapshot laggards from `qsmask` under existing locks in `rcu_gp_fqs()` or its equivalent forcing cadence, choose exactly one housekeeping steward per eligible cohort and epoch, arm a pinned high-resolution timer only on non-RT kernels with local APIC deadline support, and use a gp_seq-keyed `nudged_once` bitmap with atomic bitops to guarantee at most one outstanding non-authoritative nudge per target per GP. 6) Add tracepoints for topology-epoch publish, qspinlock bucket collision/escape, scheduler bucket commit, and RCU steward arm/cancel/expiry; expose only trace output, never a policy API. 7) Wire concrete fail-closed triggers: hotplug epoch change, CPU isolation/housekeeping update, virtualization ambiguity, alias collision, rep stall, expedited GP, PREEMPT_RT, or memory-ordering doubt all force immediate stock behavior.

2.6 Validation Plan

Run locktorture, qspinlock microbenchmarks, will-it-scale lock workloads, hackbench, netperf wake fan-out tests, rcutorture, and multi-socket database/AI serving traces on at least one x86 server with x2APIC logical clustering and one environment where x2APIC batching is unavailable so fallback behavior is exercised. Use `perf lock`, `perf c2c`, ftrace, lockstat, and scheduler tracepoints to show: (a) fewer global qspinlock queue entries and reduced remote cacheline migration for same-domain contention; (b) no change in stock unlock order and no starvation beyond the configured bounded-unfairness limits; (c) identical per-target enqueue visibility and destination pending semantics compared to stock; (d) reduced architectural send operations only when multiple target CPUs in one primitive class share one legal APIC cluster, with worst-case send count equal to baseline otherwise; (e) for Tree RCU, no quiescent state credited from delivery success, at most one outstanding nudge per target per GP via the gp_seq-keyed bitmap, and exact cancellation on GP advance or steward hotplug. Stress hotplug, CPU offline/online churn, nohz_full, housekeeping changes, rcu_nocb, paravirt guests, and PREEMPT_RT builds to confirm optimization disables cleanly. Add targeted fault injection to force representative CPU preemption/offline, bucket collisions, topology epoch flips mid-wait, and stale timer expiry to prove escape-to-stock and no stale post-GP nudges.

3. Patent Claims

  1. A Linux synchronization substrate that derives an optimization-only topology epoch from x86 CPUID 0x1F or 0x0B x2APIC hierarchy, publishes it via RCU, and enables cohort behavior only when package or explicitly enumerated die boundaries are stable across all online CPUs.
  2. An acquisition-time qspinlock cohort gate implemented with a fixed hashed sidecar keyed by lock address, wherein only one representative per topology domain may enter the stock qspinlock slowpath while same-domain peers wait outside the global queue, and wherein unlock semantics remain the unmodified stock immediate-successor behavior.
  3. The qspinlock gate of claim 2 further comprising explicit fail-closed escape conditions based on representative age, iteration count, batch-credit exhaustion, bucket collision, CPU offline, topology epoch mismatch, paravirtual spinlock presence, or PREEMPT_RT mode, such that any uncertainty drains the cohort back to stock behavior.
  4. A scheduler remote-wake batching method in which the authoritative enqueue publication and first-transition pending-state update for each destination CPU are completed exactly as in stock Linux before that CPU is added to a sender-local transient mask, the transient mask being grouped only by APIC cluster and primitive class and committed immediately after the lexical batching window ends.
  5. The batching method of claim 4 wherein an x86 arch hook may replace multiple stock per-target transport operations with one legal logical- or cluster-mode x2APIC send only when every target CPU in the mask shares identical primitive semantics and one legal destination cluster, and otherwise the stock per-target helper sequence is executed without semantic change.
  6. A Tree RCU forcing optimization in which only the existing GP forcing path reads authoritative laggard state, groups laggards by current topology cohort, and arms at most one housekeeping steward timer per cohort and grace-period sequence, the timer revalidating current gp_seq and authoritative laggard membership before issuing any non-authoritative nudge.
  7. The Tree RCU optimization of claim 6 further comprising a transient gp_seq-keyed duplicate-suppression bitmap that guarantees at most one outstanding non-authoritative notification per target CPU per grace period while contributing nothing to quiescent-state accounting, grace-period completion, or qsmask mutation.

4. Risks & Mitigations

  • Risk: topology ambiguity or drift under hotplug, BIOS quirks, or virtualization could mis-group CPUs; mitigation: consume only CPUID 0x1F/0x0B shift hierarchy, require explicit stable die enumeration before enabling die scope, rebuild snapshots from CPUHP events, compare epochs on every use, and fall back immediately on mismatch.
  • Risk: qspinlock sidecar alias collisions could serialize unrelated locks; mitigation: fixed hash entries are key-claimed by lock address, any live-key mismatch disables optimization for that attempt, and no lock ever depends on the sidecar for correctness.
  • Risk: representative gating could strand waiters when the representative is preempted or offlined; mitigation: each waiter tracks rep_gen progress against bounded iteration/time limits, offline detection, and batch-credit exhaustion, then forces drain-to-stock mode for that generation.
  • Risk: prior-art overlap with existing ttwu pending suppression; mitigation: claim only cross-target architectural send collapse across same-class, same-cluster targets after authoritative per-target state already exists, not fewer wake events or a new pending-state mechanism.
  • Risk: mask batching could mix semantically different nudge behaviors; mitigation: classify each target into concrete primitive classes mapped to existing APIs, split masks by class and policy, and force singleton stock handling for heterogeneous or uncertain targets.
  • Risk: APIC mask send may not be a real single-shot hardware operation on all x86 parts; mitigation: the arch hook is allowed to batch only when its projected send count is strictly no worse than baseline and the mask stays within one legal logical cluster, otherwise it returns false and the caller performs the stock loop.
  • Risk: Tree RCU could accidentally infer quiescent state from transport success or race GP advance; mitigation: authoritative qsmask is read only under existing locks, timer callbacks recheck current gp_seq before acting, duplicate suppression is gp-local and non-authoritative, and GP advance cancels timers and resets sidecars.
  • Risk: PREEMPT_RT or paravirt modes may invalidate assumptions about contexts and spinning; mitigation: disable qspinlock cohort gating and steward timers under PREEMPT_RT and paravirt spinlock paths, while allowing only scheduler batching paths that already use non-sleeping stock APIs in the same context.
  • Risk: extra local bookkeeping could outweigh gains on small systems; mitigation: guard every feature behind static keys, require a minimum contention or fan-out threshold, and let the arch hook reject batching unless it beats the baseline cost model.
