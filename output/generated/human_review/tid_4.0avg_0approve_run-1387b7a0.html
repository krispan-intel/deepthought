2. Technical Invention Disclosure

2.1 Problem Statement

Intel TDX guests face two coupled hot-path costs that conventional Linux drivers do not jointly solve. First, device-visible descriptors, completion records, and some staging buffers must reside in host-visible SHARED memory, so any accidental emission of kernel-private bytes, stack data, or stale metadata into those structures becomes a confidentiality failure rather than a simple bug. Second, each MMIO or hypercall doorbell can be materially more expensive in a confidential guest, so naive immediate kicks waste performance. Existing drivers optimize DMA and batching for ordinary VMs, but they do not provide a kernel-enforced memory-state contract, a transport-specific stale-completion rejection model, or a bounded and non-starving notify-deferral rule that preserves virtio split-ring ordering, packed-ring wrap semantics, and NVMe submission/completion ordering. The problem is to reduce TDX queue overhead while proving that no PRIVATE alias is published as SHARED, no malformed or replayed completion is accepted as trusted state, no DMA lease is reclaimed early, and no kick deferral can sleep, allocate, or starve progress in atomic or softirq paths.

2.2 Prior-Art Gap

Known mechanisms each cover only a fragment. dma_pool and dma_alloc_coherent provide aligned DMA memory but do not prove that a queue arena is permanently SHARED for its active lifetime, PFN-tracked, alias-restricted, and separated from private metadata. page_pool, page recycling, and bounce-buffer reuse improve throughput but do not require completion-bound SHARED-to-PRIVATE conversion and scrub before reuse. Event-idx, vring_need_event, xmit_more, blk-mq plugging, interrupt moderation, and napi batching reduce notifications but do not define a virtio-core-owned, TDX-specific, non-extendable deadline contract with exact must-attempt boundaries and atomic-safe expiry handling. Standard shadow rings and templating may reduce stores, but they do not define a field-whitelist matrix that constrains every emitted descriptor field by bounds, endian, and provenance, nor do they ban memcpy of driver-private structs into shared rings. Finally, existing queue logic typically trusts transport identifiers such as descriptor id or CID as sufficient for completion association, whereas a hostile host can replay stale used elements or CQEs across wrap, reset, or slot reuse unless the guest maintains private generations, queue epochs, and transport-specific acceptance rules.

2.3 Proposed Invention

The invention is a Linux helper library, tdx_ioq, with five cooperating subcontracts. First, a tdx_shared_arena allocator creates per-queue page-backed arenas only from direct-map-backed struct page allocations or coherent queue memory, converts those pages to SHARED before first publication using the architecture's shared-memory primitive, records the PFNs in a private ownership table, and forbids vmalloc-backed memory, highmem pages requiring transient kmap aliases, io_remap aliases, or emission of raw kernel virtual addresses; pinned user pages may be DMA payloads, but only DMA addresses returned by the DMA API may be written into shared descriptors. Second, a descriptor-section whitelist matrix defines each transport-visible structure as a list of fields with offset, width, endianness, alignment rule, and provenance enum such as constant, DMA address from dma_map_sg or dma_map_page, approved shared offset within the same arena, bounded length, generation echo bits, or transport flags; indirect tables are a separate descriptor class with bounded element count and per-entry encoding, and the contract explicitly forbids memcpy of driver-private structs into SHARED memory. Third, each queue slot has private metadata including slot_state, 64-bit generation, queue_epoch, dma_lease state, and an optional secret-derived cookie. Completion acceptance then runs in one of three modes: echo-capable mode for NVMe or vhost-like transports that echo a driver-controlled token, requiring id plus echoed token plus current generation or queue_epoch match; ordered-used mode for virtio split rings, requiring monotonic used-head progression, an inflight slot for the returned descriptor id, and a generation that has not been retired; and packed-ring mode, requiring inflight slot state, descriptor id consistency, and the expected wrap or phase relation from the packed ring metadata. When no extra echoed bits exist, the fallback remains safe because stale host replay is rejected on slot state, generation, or queue_epoch mismatch, with residual failure mode limited to drop, timeout, or queue reset rather than silent corruption. Fourth, DMA staging pages are treated as leases: completion processing first validates freshness, then performs dma_unmap_* or dma_sync_* in DMA-API-correct order, then scrubs the SHARED bytes that carried request-specific data, then converts the page back to PRIVATE if it will re-enter a private pool, and only after conversion performs final memzero_explicit before reuse. Fifth, kick deferral is implemented as a per-queue epoch state machine in virtio-core or the transport adapter. The first deferrable publish in an idle queue sets a fixed deadline from CLOCK_MONOTONIC via hrtimer; later publishes in the same pending epoch may piggyback but cannot extend that deadline. Must-attempt boundaries are explicit and driver-visible: queue drain or stop, freeze or suspend preparation, reset initiation, descriptor or indirect-table exhaustion, xmit_more false flush points, NAPI poll end after queue publication, blk plug flush or blk_mq_commit_rqs boundaries, packed-ring wrap-sensitive publication, feature-dependent barrier transitions, and return paths that promise submission progress to upper layers. On expiry, hrtimer queues irq_work rather than sleeping work; irq_work attempts the kick if the queue lock is available, otherwise it sets a force bit that obligates the current lock owner to attempt before unlock. The actual notify predicate remains the transport's own logic: virtqueue_kick_prepare, vring_need_event, or the NVMe doorbell wrapper decides whether an actual notify is needed. Deferral only chooses when that authoritative predicate is consulted.

2.4 Architecture

Per-queue objects:
1. tdx_shared_arena { shared_pages[], pfn_xarray, shared_offsets, allowed_desc_classes }
2. tdx_slot_meta[queue_depth] { generation64, queue_epoch, state, dma_lease, optional_cookie }
3. tdx_kick_epoch { state, deadline_ktime, pending_reason_bits, force_bit, hrtimer, irq_work }

Fast path:
private skb/bio/cmd
   -> private scratch parse
   -> whitelist encoder matrix validates each field
   -> write only approved bytes into SHARED descriptor section or indirect table
   -> dma_wmb()/virtio_wmb()/wmb as transport requires
   -> publish ring index or SQ tail shadow
   -> tdx_kick_maybe_defer(reason)
   -> virtqueue_kick_prepare()/need_notify() remains authoritative at attempt time

Completion path:
used elem / packed used bit / NVMe CQE
   -> freshness check against slot_meta + queue_epoch + transport mode
   -> dma_unmap_* or dma_sync_*
   -> scrub SHARED request bytes
   -> if page returns to private pool: set_memory_encrypted() -> memzero_explicit()
   -> retire generation and free slot

Kick state machine:
IDLE --first deferrable publish--> PENDING(deadline fixed once)
PENDING --must-attempt reason--> ATTEMPT
PENDING --hrtimer expiry--> ATTEMPT
ATTEMPT --authoritative need-notify false--> IDLE
ATTEMPT --notify issued--> IDLE
ANY --reset/freeze/unplug/broken--> BROKEN_OR_QUIESCED(cancel timer, increment queue_epoch, reject stale completions)

Concurrency rules:
- Shared arena contents are written only under the queue's existing producer lock or equivalent serialization.
- tdx_kick_epoch is updated under the same queue lock, except hrtimer and irq_work may set force_bit atomically.
- hrtimer and irq_work never allocate and never sleep.
- If expiry races with a held queue lock, the lock owner must observe force_bit and run the must-attempt hook before unlock.
- The timer uses CLOCK_MONOTONIC; PM freeze path forces or cancels pending epochs before suspend so suspended time cannot silently extend a deferral window.

Memory-state rules:
- Arena pages are page-backed only, converted to SHARED before publication, never emitted through vmalloc or temporary kmap aliases, and never mixed with private metadata.
- CONFIG_VMAP_STACK and highmem are handled by prohibition: stack objects and transiently mapped highmem pages cannot be encoded directly into SHARED descriptors; they must first become DMA-mapped payloads or be copied through private scratch to approved shared fields.

2.5 Implementation Plan

1. Add a new core helper with struct tdx_shared_arena, struct tdx_slot_meta, struct tdx_kick_epoch, and enum tdx_completion_mode plus helpers tdx_ioq_encode_desc(), tdx_ioq_publish_complete(), tdx_ioq_maybe_defer_kick(), tdx_ioq_force_attempt(), and tdx_ioq_complete(). 2. Arena allocation path: allocate queue pages from the page allocator or coherent queue memory, convert with set_memory_decrypted() or architecture wrapper before queue enable, register PFNs in an xarray keyed by page frame, and reject vmalloc, highmem, kmap-local, io_remap, or alternate alias registration for those PFNs. 3. Virtio split-ring integration: wrap descriptor population in drivers/virtio/virtio_ring.c at virtqueue_add_split(), place whitelist encoding before existing virtio_wmb(), and route the post-publication path through tdx_ioq_maybe_defer_kick(); preserve virtqueue_kick_prepare() as the only place that decides if notify is needed. Completion-side hooks attach to the split-ring get-buf path so used elements are validated against slot_meta before DMA lease retirement. 4. Virtio packed-ring integration: mirror the split-ring integration at virtqueue_add_packed() and packed completion retrieval, with explicit storage of expected wrap or phase in slot_meta; do not let deferral reorder any packed availability or wrap update relative to existing barriers. 5. Driver-visible must-attempt hints live at subsystem boundaries rather than in ad hoc heuristics: virtio-net passes xmit_more false and NAPI poll-end reasons, virtio-blk passes blk plug flush and blk_mq_commit_rqs boundaries, freeze or stop paths pass quiesce reasons, and queue exhaustion paths pass immediate flush reasons. State remains in virtio-core, while drivers provide only boundary hints. 6. NVMe embodiment: store the same tdx_slot_meta and tdx_kick_epoch in struct nvme_queue; encode SQEs through the whitelist layer, doorbell through tdx_ioq_maybe_defer_kick(), and validate CQEs in nvme_process_cq() using CID plus phase plus generation, with an echo-capable mode when enough command-id bits are reserved. 7. Expiry handling: initialize hrtimer and irq_work at queue creation; on timer expiry, queue irq_work to avoid sleeping workqueues; irq_work attempts a kick via spin_trylock or sets force_bit for the current lock owner to honor before unlock. 8. Reset, device unplug, and PM freeze paths increment queue_epoch, cancel active timers, clear pending deferral state, and mark outstanding slots as failed so stale completions after reset cannot be accepted. 9. Runtime gating: enable only when TDX guest detection is true, the transport declares atomic-safe kick capability and a max_defer_ns, and the queue is not flagged latency-critical; otherwise execute the unmodified immediate-kick path.

2.6 Validation Plan

Validate in four layers. Layer 1, KUnit and fault-injection tests for the descriptor whitelist matrix: reject out-of-bounds fields, bad endianness declarations, non-DMA-derived addresses, oversized indirect tables, and memcpy attempts; verify approved fields encode identically to existing virtio or NVMe layouts. Layer 2, concurrency and correctness tests: race hrtimer expiry with producer lock hold, queue reset, freeze, and unplug; prove no sleeping or allocation occurs in kick paths; verify deadline is armed once and cannot be extended while pending; verify force_bit causes a must-attempt before unlock. Layer 3, transport semantics tests: virtio split event-idx paths, packed wrap transitions, virtio-net xmit_more batches, NAPI poll-end flush, virtio-blk plug flush, and NVMe phase-tag handling; inject replayed used elements or CQEs across slot reuse and reset to show that slot generation or queue_epoch rejection fires. Layer 4, memory-state and DMA tests: use KASAN, KFENCE, and DMA-API debugging to confirm no early unmap, no reuse before completion retirement, successful SHARED-to-PRIVATE conversion before private-pool return, and no arena PFN alias registration. Performance evaluation should compare TDX guest throughput, p99 latency, and kick-attempt counts against baseline virtio and NVMe drivers over multiple defer windows, with correctness fallback enabled on any timeout, reset storm, or transport that lacks the required capability bits.

4. Risks & Mitigations

  • Prior-art overlap with generic batching or event suppression: mitigate by claiming the narrower combination of TDX-only gating, PFN-tracked SHARED arenas, whitelist field provenance, transport-specific freshness validation, and a non-extendable bounded kick-attempt state machine that leaves virtqueue_kick_prepare authoritative.
  • Atomic-path deadlock or latency regression: mitigate by preallocating hrtimer and irq_work at queue creation, forbidding workqueue use on forced-attempt paths, updating deferral state under existing queue locks, and using a force-bit plus pre-unlock must-attempt rule when timer expiry races with a held lock.
  • Incorrect memory-state handling or PRIVATE alias leakage: mitigate by allowing only page-backed arena allocations, tracking arena PFNs, rejecting vmalloc and highmem aliases, prohibiting direct emission of stack or driver-private structures, and adding debug assertions for alias registration and shared-page ownership.
  • Transport mismatch or insufficient echoed bits for freshness: mitigate by defining explicit completion modes per transport, using ordered-used mode for virtio when only descriptor ids and used ordering are visible, using echo-capable mode when NVMe or vhost-like backends return driver-controlled bits, and falling back to drop or reset rather than accepting ambiguous completions.
  • Suspend, reset, or unplug races with pending deferral: mitigate by forcing or canceling pending epochs during freeze or reset entry, incrementing queue_epoch on reset or unplug, canceling active timers, and rejecting post-reset completions that reference retired epochs.
  • Performance regression from over-deferral or starvation against xmit_more or blk plugging: mitigate by making the deadline non-extendable once armed, enumerating mandatory flush boundaries, constraining per-transport max_defer_ns, and auto-disabling the feature on latency-critical or timeout-prone queues.
  • Host-driven malformed completion or replay attack: mitigate by validating slot state, generation, queue epoch, and transport ordering before DMA lease retirement, quarantining malformed queues, and making the failure mode an explicit drop, timeout, or reset instead of silent data corruption.
