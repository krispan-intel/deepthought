2. Technical Invention Disclosure

2.1 Problem Statement

Linux interrupt steering on large x86 servers is limited by a three-way conflict. First, interrupt locality shifts quickly across cores, LLC domains, and sockets as network queues, block completions, and virtual interrupts rebalance. Second, legal placement is not just a CPU mask question: it is constrained by online state, boot-time `irqaffinity=` defaults, explicit user affinity requests, housekeeping-versus-isolated CPU policy, optional cpuset-derived policy masks, and for x86 MSI/MSI-X by current `irq_matrix` vector scarcity and remap cost. Third, correctness is brittle because IRQ descriptors can be reconfigured or freed while CPU hotplug, isolation updates, and vector remaps occur concurrently. A useful in-kernel optimizer therefore must satisfy all of the following simultaneously: strict O(1) hardirq overhead under adversarial IRQ IDs; zero affinity writes in atomic context; no extension of x86 vector-allocation critical sections; exact separation of requested versus effective affinity; no use-after-free on sparse IRQ descriptors; no conflict with `IRQD_AFFINITY_MANAGED`; and deterministic behavior under `-EBUSY`, `-EINVAL`, hotplug, or teardown races.

2.2 Prior-Art Gap

Known solutions fail because each omits at least one required element of the full bridge. User-space balancers such as irqbalance observe too late and cannot sample from hardirq context with bounded constant cost. Conventional in-kernel affinity helpers can request moves but do not define a fixed-probe hardirq sketch, an immutable epochal IRQxCPU heatmap, or a vector-pressure-aware scoring algebra tied to x86 `irq_matrix` state. Sketching literature covers Count-Min or heavy-hitter approximations, but not a kernel IRQ-specific design with exact adversarial bounds, per-CPU false-sharing avoidance, saturating epoch semantics, and affinity legality integration. Standard seqcount/RCU retry patterns describe how to recheck a changing world, but not a specific state machine that must simultaneously validate topology, isolation, cpuset-policy generation, vector allocation epoch, immutable policy publication, and per-descriptor lifetime generation before invoking irq/core setters. AMX prior art would treat matrix multiplication as a generic speedup; here the matrix shapes and semantics are part of the invention because `E x T` performs topology-equivalent neighborhood convolution over the frozen IRQxCPU heatmap and `R x Q` injects vector scarcity, allocation-failure history, and remap penalty directly into placement scores. The novelty boundary is therefore the combined contract, not any single ingredient.

2.3 Proposed Invention

Introduce an `irq_amx_bridge` subsystem with a bridge-local control plane, per-CPU collector banks, a freezer, an AMX scorer, and a descriptor-safe commit worker. Hardirq COLLECT uses `struct irq_amx_slot { u16 tag; u16 cnt; }`, `struct irq_amx_bucket { struct irq_amx_slot way[4]; u8 rr; u8 pad[7]; } __aligned(32)`, and `struct irq_amx_bank { struct irq_amx_bucket b[256]; }`; each CPU owns two banks in `__percpu` storage, each bank is about 4 KiB plus metadata, and both banks are aligned to 128 bytes to prevent inter-CPU cacheline bouncing. On each interrupt, the collector reads the current bank pointer from `this_cpu_ptr`, computes exactly two multiply-shift hashes `h1` and `h2` with per-boot random seeds, computes a 16-bit nonzero tag `tag = fold16(irq ^ tag_seed) | 1`, compares the four ways in bucket `h1` and then the four ways in bucket `h2`, saturating the matching `cnt` at `0xffff` if a tag matches, otherwise inserting into the first empty way found by fixed comparisons, otherwise replacing the round-robin victim chosen by `rr` in a predetermined bucket. The collector therefore performs exactly two hashes, eight tag/empty comparisons, one `rr` update, and at most two stores; there are no data-dependent loops, no remote cacheline writes, and no IRQ-number-dependent memory fanout. The design budget is less than 64 retired instructions and less than 80 cycles when the per-CPU bank is L1-hot, with worst-case bounded even for sparse or adversarial IRQ numbers. Saturated counters set a per-bank overflow bit so the scorer treats them as lower bounds rather than silently wrapping. FREEZE runs on a periodic timer or on a vector-pressure threshold crossing. The coordinator samples begin values of `topology_seq`, `isolation_seq`, `cpuset_seq`, `policy_seq`, and `vector_seq`, then issues `smp_call_function_many()` to the participating online CPUs. Each CPU callback performs only a local bank flip: `old = active_bank; smp_store_release(active_bank, old ^ 1); bank_epoch[old] = freeze_epoch;` and returns. Because each CPU writes only its own state, FREEZE adds no shared lock to hardirq paths and no new cacheline ping-pong. After the callbacks complete, the coordinator reads the same seqcounts with retry checks; if any are inconsistent or changed during the capture window, the entire snapshot is discarded and retried once, after which the epoch becomes a deterministic no-op if consistency still fails. Vector and legality state is captured without burdening fast paths. `vector_seq` is incremented only in existing x86 vector allocation/free slow paths, alongside bridge-local shadow fields such as `free_vectors`, `reserved_managed`, `recent_alloc_failures`, and `remap_penalty` per CPU. `topology_seq` increments on CPU hotplug and topology rebuild callbacks. `isolation_seq` increments when housekeeping or `nohz_full`/`isolcpus` policy changes. `cpuset_seq` increments on cpuset effective mask changes; by default cpuset is observe-only to preserve baseline kernel semantics, but an admin policy bit can make the bridge intersect target masks with cpuset-derived service masks. `policy_seq` increments when a new immutable RCU-published policy object is installed. Per-IRQ `desc_gen` lives in a bridge sidecar `irq_amx_desc_meta`, stored in an RCU-protected xarray keyed by IRQ number and refcounted so the bridge never holds a bare `irq_desc *` across sleeping stages. The sidecar generation is incremented only in slow paths such as descriptor registration/unregistration, affinity-policy metadata change, and bridge-visible managed-state transitions; increments do not touch the interrupt fast path. COMPACT selects at most `K = 32` candidate IRQs per NUMA node from frozen banks by fixed-cost extraction over the immutable 256x4 slots per CPU and chooses at most `M = 8` currently legal target CPUs per node. Legality precedence is explicit: mandatory filters are `cpu_online_mask`, irqchip/domain-supported mask, bridge housekeeping mask derived from `isolcpus` and `nohz_full` unless the policy explicitly allows isolated targets, boot-time `irqaffinity=` or `default_smp_affinity` for IRQs with no explicit user mask, and any stronger explicit user-requested mask already attached to the IRQ. Optional cpuset intersection applies only when `policy.respect_cpuset = 1`. The bridge may only narrow a legal mask; it may never widen beyond kernel or user ownership. Requested affinity and effective affinity remain separate: scoring uses the requested-legal mask, and post-apply verification reads effective affinity to learn chip-side clamping. SCORE is matrix-native and bounded. For each epoch, build `E[K,M]`, an int8 or int16 heatmap whose entry `E[k,m]` is the frozen interrupt count for candidate IRQ `k` observed on candidate CPU `m`. Build `T[M,M]`, a signed topology kernel where same CPU = +8, SMT sibling = +6, same LLC = +4, same NUMA = +2, remote socket = -8, and illegal target = a large negative mask value. Compute `L = E x T`; this is a matrix-equivalent topology convolution that spreads each IRQ's historical activity over nearby CPUs according to actual x86 locality. Build `R[K,4]` with per-IRQ steering weights `[rate, burstiness, vector_demand_class, hysteresis_weight]`. Build `Q[4,M]` with per-CPU vector and policy features `[normalized_free_vectors, allocation_failure_penalty, remap_penalty, housekeeping_or_isolation_penalty]`. Compute `V = R x Q`. Form `H[K,M]`, a direct hysteresis matrix that penalizes movement away from the currently effective CPU, and `U[K,M]`, a legality mask with zero for legal cells and a very negative value for illegal cells. Final score is `S = alpha*L + beta*V - gamma*H + U`, all in bounded signed integer arithmetic. Intel AMX tile dot-product instructions execute the `E x T` and `R x Q` multiplies inside a housekeeping kthread; topology smoothing and vector scarcity are therefore not implementation detail but the scoring primitive itself. If AMX is unavailable, disabled by XFD, or not exposed to a guest kernel, the exact same algebra runs in an AVX2 or scalar backend. The kernel AMX execution contract is explicit. Only the dedicated scorer kthread may enter XTILE state. It checks CPU feature exposure for AMX tile state and xsave support at thread start, acquires kernel FPU/XTILE ownership only in process context, never in hardirq or IRQ-disabled sections, and inserts preemption points between tile batches. First-use XTILE enablement and any XFD clear happen once per scorer thread; if XTILE state cannot be enabled, the bridge permanently marks the node backend as non-AMX and continues with the fallback scorer. No AMX state is carried through COMMIT. COMMIT operates as a deterministic state machine: `PLAN(epoch, score_tuple) -> REVALIDATE -> APPLY -> VERIFY` with at most one retry recompute and then no-op. For each planned move, the worker first RCU-dereferences the sidecar meta, takes a refcount, reacquires the current descriptor lookup, and checks that `desc_gen`, `topology_seq`, `isolation_seq`, `cpuset_seq`, `policy_seq`, and `vector_seq` still match the planning tuple. It rebuilds the current legal mask from live state and drops the move if the proposed target is no longer legal. If the IRQ is unmanaged, it calls the existing irq/core affinity setter path in process context only; it never open-codes APIC or vector changes. If the setter returns `-EBUSY`, `-EINVAL`, or the descriptor has disappeared, the move becomes a no-op for the epoch. The worker then reads effective affinity; if the chip/domain clamps the request, that observed effective mask is fed back into the next epoch's hysteresis state. If the IRQ carries `IRQD_AFFINITY_MANAGED`, the worker never calls a direct affinity setter. Instead the rule is exact: if bridge policy mode is `managed_noop`, record score and emit tracepoint only; if mode is `managed_hint` and there is no driver-owned immutable hint conflict, publish a clamped `irq_set_affinity_hint()` suggestion limited to the current legal housekeeping mask, but still perform no direct move. Thus the bridge cannot fight managed affinity updates or remapped vectors.

2.4 Architecture

Subsystem and state machine:

`hardirq CPU i`
`  handle_irq_event_percpu()`
`    -> irq_amx_collect(irq)`
`       bank = this_cpu(active_bank)`
`       h1 = hash1(irq); h2 = hash2(irq)`
`       probe 4 ways in b[h1], then 4 ways in b[h2]`
`       saturate or fixed-policy replace`
`       // no locks, no allocations, no affinity writes`

`per-CPU state`
`  bank[2] : 256 buckets x 4 ways`
`  active_bank : 0|1`
`  bank_epoch[2]`
`  overflow_bitmap`

`coordinator on housekeeping CPU`
`  read_seqbegin(topology_seq, isolation_seq, cpuset_seq, policy_seq, vector_seq)`
`  smp_call_function_many(observation_cpus, flip_cb)`
`  if any seq changed or retried -> discard epoch, retry once`
`  compact frozen banks -> E[K,M]`
`  snapshot vector shadow -> Q[4,M]`
`  snapshot legality + current effective affinity`

`scorer kthread (process context only)`
`  if AMX available:`
`      L = E x T        // topology convolution`
`      V = R x Q        // vector pressure and remap cost`
`      S = alpha*L + beta*V - gamma*H + U`
`  else:`
`      same algebra via AVX2/scalar`
`  produce ranked plan list`

`commit worker`
`  for each ranked move:`
`      meta = xa_load_rcu(irq)`
`      refcount_inc(meta)`
`      revalidate(desc_gen, topology_seq, isolation_seq, cpuset_seq, policy_seq, vector_seq)`
`      rebuild legal mask`
`      if managed -> hint-or-no-op rule`
`      else irq/core affinity setter`
`      read effective_affinity`
`      record success, clamp, or deterministic no-op`

Generation sources:
`  topology_seq  <- CPU hotplug / topology rebuild callbacks`
`  isolation_seq <- housekeeping, isolcpus, nohz_full updates`
`  cpuset_seq    <- cpuset effective mask changes`
`  policy_seq    <- immutable RCU policy publication`
`  vector_seq    <- x86 irq_matrix alloc/free slow paths`
`  desc_gen      <- bridge sidecar lifetime/metadata changes`

UAF avoidance:
`  irq number -> RCU xarray -> irq_amx_desc_meta(refcounted)`
`  score/commit store irq numbers + meta generation, never bare irq_desc pointers across sleep`

Commit invariants:
`  APPLY is legal only if all generation values still match and target is still within the rebuilt legal mask.`
`  Retry bound = 1 recompute attempt per epoch; second conflict becomes explicit no-op.`

2.5 Implementation Plan

1. Add `irq_amx_bridge` core objects: per-CPU double banks, bridge policy object, generation counters, and `irq_amx_desc_meta` sidecar xarray with RCU plus refcount semantics. 2. Hook COLLECT into the generic IRQ handling path at a point that already knows the final IRQ number and CPU, and keep the collector fully self-contained in `__percpu` memory. 3. Add CPU hotplug, housekeeping/isolation, cpuset, and policy-publication notifiers that increment bridge seqcounts only on slow-path events. 4. Add bridge-local shadow accounting near x86 vector allocation/free slow paths so `vector_seq` and per-CPU vector-pressure fields update without changing the hardirq path. 5. Implement FREEZE with synchronous per-CPU bank flip callbacks and a single retry rule for inconsistent seqcount windows. 6. Implement bounded COMPACT that extracts top-K candidate IRQs and top-M legal CPUs per NUMA node from frozen banks. 7. Implement SCORE backends sharing the same matrices and integer arithmetic: scalar reference backend first, then AVX2, then AMX tile backend. The AMX backend must be limited to a dedicated housekeeping kthread that enters kernel FPU/XTILE sections only in process context, clears or enables XFD once, and never runs with IRQs disabled. 8. Implement COMMIT as a workqueue or kthread worker that revalidates generations, rebuilds the live legal mask, and delegates all actual affinity changes to existing irq/core affinity setter APIs. 9. Implement the exact managed-IRQ policy modes `noop` and `hint`, defaulting to `noop`; never call a direct setter for `IRQD_AFFINITY_MANAGED`. 10. Expose read-only observability through tracepoints and `seq_file`, and expose control only through an init-namespace generic-netlink family requiring `CAP_SYS_ADMIN` plus an LSM authorization hook. Policy objects are immutable after publication, replaced RCU-style, and rate-limited by a token bucket to prevent policy spam or FREEZE storms. 11. Reject control-plane requests that widen legal masks, target forbidden isolated CPUs without explicit opt-in, or exceed a configured freeze frequency.

4. Risks & Mitigations

  • Risk: sketch collisions can mis-rank low-volume IRQs. Mitigation: use fixed two-hash four-way layout with overflow bits, tune K and M conservatively, and restrict COMPACT to high-signal candidates while retaining scalar validation counters for top entries.
  • Risk: AMX kernel enablement may be unavailable, expensive, or disallowed in guests. Mitigation: confine XTILE use to one scorer kthread, enable once per thread, never use AMX in atomic context, and provide AVX2/scalar backends with identical score semantics.
  • Risk: adding generation counters could create cacheline contention. Mitigation: increment `topology_seq`, `isolation_seq`, `cpuset_seq`, `policy_seq`, `vector_seq`, and `desc_gen` only in slow-path notifiers or allocator paths already serialized by existing locks; never touch them from the collector fast path.
  • Risk: sparse IRQ descriptor teardown could cause use-after-free. Mitigation: store bridge metadata in a refcounted RCU sidecar xarray, carry IRQ numbers plus `desc_gen` across sleeping stages, and reacquire live descriptor state only during REVALIDATE/APPLY.
  • Risk: bridge policy could violate kernel ownership or user expectations. Mitigation: make legality precedence one-way narrowing only, default cpuset handling to observe-only, preserve explicit user-requested masks, and verify effective affinity after every applied move.
  • Risk: managed MSI/MSI-X IRQs could be disturbed. Mitigation: branch on `IRQD_AFFINITY_MANAGED` before any apply, default to no-op, and allow only optional clamped hint publication with no direct setter call.
  • Risk: control-plane abuse could trigger constant FREEZE/SCORE churn or steering toward isolated CPUs. Mitigation: use init-namespace-only generic netlink, require `CAP_SYS_ADMIN` plus LSM authorization, enforce token-bucket rate limiting, reject policies that widen masks or violate isolation constraints, and keep only one pending immutable policy object.
  • Risk: `irq_set_affinity()` may return `-EBUSY` or `-EINVAL` under remap or domain constraints. Mitigation: treat such results as first-class bounded outcomes, record them in feedback state, and convert repeated refusals into score penalties on subsequent epochs rather than retry loops.
