1. Executive Summary

One-Liner

A driver-visible, domain-scoped DMA lease object that only accepts page-granular buffers with provable exclusive ownership, prepares them in sleepable context by pinning or normalizing eligible pages, converts those pages from TDX or CoCo private to shared using the existing x86 shared-memory transition path, binds that shared window to a streaming DMA mapping and optional schema-validated descriptor publication sequence, permits only IRQ-safe completion marking in atomic context, and restores privacy solely in deferred process context after required dma_sync, DMA unmap, device quiesce, and IOMMU or device-translation drain semantics have completed.
Novelty: The invention is not a generic refcount plus workqueue cleanup pattern. Its novelty is a single kernel object that enforces, in one state machine, four constraints that current Linux drivers handle only piecemeal and oft
Feasibility: The design is implementable with existing Linux mechanisms plus a small set of new glue hooks. It uses real primitives and real context rules: pin_user_pages* with FOLL_PIN and opt

2. Technical Invention Disclosure

2.1 Problem Statement

In an x86 confidential guest such as Intel TDX, devices cannot safely DMA into guest-private pages because device-side and hypervisor-mediated address translation only observe shared memory. Linux drivers therefore fall back to permanently shared rings, shared bounce pools, or ad hoc calls around CoCo page conversion and dma_map or dma_unmap. Those patterns routinely leave six hard gaps. First, buffer eligibility is underspecified: many drafts claim support for arbitrary DMA buffers even though slab allocations, vmalloc ranges, page fragments, file-backed folios, DAX mappings, shared-anon pages, KSM pages, hugetlb pages, and partially covered compound pages are not safely convertible in place. Second, user-page handling is often kernel-incorrect because it blurs get_user_pages and pin_user_pages semantics, ignores FOLL_PIN and FOLL_LONGTERM requirements, and cannot prove exclusive ownership against COW, KSM, or secondary mappings. Third, DMA visibility is often described too vaguely, especially for non-coherent devices where dma_sync_for_device and dma_sync_for_cpu points differ by direction and where descriptor publication must use dma_wmb rather than a generic CPU-only barrier. Fourth, simplistic 'dma_unmap then re-private' claims ignore IOMMU and device translation caches such as ATS, PASID, PRI, or SVA that can leave the device able to use a stale translation after unmap. Fifth, teardown paths rarely specify how outstanding shared buffers are tracked, how reset forces failure, or how lease objects survive IRQ completions racing driver detach. Sixth, descriptor sharing proposals frequently claim a 'no-pointer' rule without defining whether IOVA fields are permitted, how schema validation works, how alignment and endianness are enforced, or how TOCTOU is prevented between validation and doorbell. The result is a security-sensitive area handled today as driver folklore rather than as an enforceable kernel protocol.

2.2 Prior-Art Gap

Existing Linux primitives each solve only one slice of the problem. dma_map, dma_unmap, and dma_sync define device-visibility rules for already-shareable memory, but they do not determine whether a buffer is legally convertible from private to shared under CoCo, nor do they bind re-private to teardown-safe deferred retirement. dma_fence orders asynchronous completion between producers and consumers, but it does not own pages, perform dma_unmap, wait for IOTLB or device-translation drain, or invoke x86 CoCo page-state transitions. dma-buf and scatterlists describe shared buffers and extents, but they intentionally do not enforce whole-page exclusivity, anon-exclusive user pinning, or confidential-memory conversion legality. IOMMUFD and SVA-related uAPIs manage user-visible address spaces and PASID attachment, not per-request kernel DMA exposure windows with mandatory re-private. devm_* and ordinary workqueue teardown can free software objects on detach, but they do not provide a cross-subsystem guarantee that a page cannot be returned to its allocator until DMA unmap, non-coherent sync, translation drain, and CoCo re-private all succeed. The invention differs by introducing a centralized kernel lease object whose prepare, publish, complete, abort, retire, and drain entry points are the only paths through which an opt-in driver can expose TDX-shared memory for streaming DMA; the core wraps both the CoCo transition and the DMA lifetime and records the exposure in a domain-tracked xarray so that reset, timeout, detach, and unload all converge on the same forced-retirement mechanism.

2.3 Proposed Invention

The invention introduces a new optional kernel library centered on two objects: struct dma_lease_domain and struct dma_lease. A dma_lease_domain is created per DMA endpoint or per tightly coupled device group by driver code in probe or open paths. It stores a struct device pointer, a spinlock_t protecting domain flags and teardown state, an xarray of outstanding leases keyed by a 64-bit lease_id, a workqueue_struct for retirement, a wait_queue_head_t for drain waiters, refcounted domain lifetime, and backend tables struct dma_lease_coco_ops and struct dma_lease_iommu_ops. A dma_lease stores the normalized page list, sg_table, direction, mapped IOVA or scatterlist, state bits in atomic_long_t, refcount_t refs, RCU publication node, work_struct retire_work, timeout metadata, optional driver cookie, and optional shadow descriptor slot. The API surface is explicit and enforceable. A new helper dma_lease_domain_create(dev, flags, ops) creates the domain and records whether coherent descriptors are used, whether user memory is permitted, and whether strict IOMMU drain is required. dma_lease_prepare(domain, req, &lease) may sleep and is the only entry point that can accept a buffer. req identifies the source type as kernel pages, page_pool full-page object, sg_table, or user anonymous mapping; kmalloc, vmalloc, module, and arbitrary virtual ranges are not accepted source types. The prepare path classifies the buffer. Order-0 pages are accepted directly if the offset and length preserve whole-base-page exclusivity. page_pool memory is accepted only when the entire base page or entire folio belongs to the page_pool object, with no fragment sharing. Compound pages or THP-backed folios are accepted only when the request spans every base page in the folio and the folio can be split in a sleepable MM path; otherwise prepare fails with -ERANGE or falls back to a domain bounce allocation. User memory is accepted only via pin_user_pages or pin_user_pages_fast using FOLL_PIN and access-direction flags, and FOLL_LONGTERM is added whenever the caller declares the lease may survive beyond a short bounded window or across indefinite wait conditions. The user-page path rejects VM_SHARED, file-backed, DAX, shmem, hugetlbfs, migrateable device-private, PFNMAP, MIXEDMAP, KSM, and non-anonymous pages. It also requires that COW be broken before publication and that anon exclusivity be provable, for example by PageAnonExclusive or an equivalent anon-exclusive state check plus mapcount validation. If exclusivity is not provable, prepare returns -EAGAIN or bounces. After classification, prepare normalizes the page array and invokes the x86 CoCo backend. On x86 TDX, the lease does not invent a new architectural transition; it calls the existing guest shared-memory transition path backing set_memory_decrypted or equivalent page-range CoCo helpers on the final page-aligned PFN range. The backend is responsible for any required direct-map or alias maintenance. The lease core rejects pages with writable aliases outside lease control. Successful conversion transitions the lease state from NEW to SHARED_PREPARED. The core then performs dma_map_page_attrs or dma_map_sgtable for streaming DMA and records the returned IOVA or sg DMA addresses. If conversion succeeded but mapping fails, rollback calls the matching CoCo re-private path before prepare returns an error. Descriptor publication is optional and separately typed. A new descriptor_schema object defines every shared field by offset, width, alignment, endianness, and semantic type such as U8, U16, U32, U64, ENUM, BITFIELD, LEASE_COOKIE, or DEVICE_IOVA. DEVICE_IOVA is the only pointer-like field type allowed, and it may only reference an active IOVA generated for the same lease under an isolated IOMMU domain. Raw physical addresses, PFNs, struct page pointers, kernel virtual addresses, kernel object addresses, and offsets relative to kernel-managed bases are invalid schema types. The driver never validates an in-place shared descriptor. Instead, dma_lease_publish_desc(lease, schema, shadow_desc, mmio_db) validates the CPU-private shadow descriptor, packs fields with cpu_to_leXX or cpu_to_beXX and put_unaligned helpers into a core-owned shared slot, marks that slot immutable, and only then transfers ownership to the device. For coherent descriptor memory, the publication sequence is write descriptor payload, dma_wmb, set the valid or owner bit, dma_wmb, then writel to the MMIO doorbell. For non-coherent descriptor memory, the sequence additionally requires dma_sync_single_for_device or dma_sync_sgtable_for_device before the first dma_wmb. If the driver uses writel_relaxed inside a lock protecting a multi-producer ring, mmiowb is required before unlocking. This makes the ordering claim concrete and Linux-portable. The lease state machine is strict: NEW -> ELIGIBLE -> SHARED_PREPARED -> DMA_MAPPED -> DESC_PUBLISHED -> INFLIGHT -> COMPLETED or FAILED -> QUIESCED -> DMA_UNMAPPED -> IOTLB_DRAINED -> REPRIVATE -> RETIRED. Only process context may execute ELIGIBLE through DESC_PUBLISHED and DMA_UNMAPPED through RETIRED, because those stages may sleep. Hardirq or softirq context may execute only dma_lease_complete(lease_id, status) or dma_lease_fail(lease_id, status), which perform kref_get_unless_zero under RCU lookup, atomically transition INFLIGHT to COMPLETED or FAILED with cmpxchg on state bits, and queue retire_work. Re-private is impossible from IRQ context because no IRQ-visible API calls the CoCo backend. Domain teardown sets a DYING flag under spinlock, rejects new prepare calls with -EBUSY, walks the xarray marking live leases FORCED_FAIL, and queues retirement for each. The lease object is removed from fast lookup only after the final retire path, and memory is freed only after refcount reaches zero and synchronize_rcu has completed, so late IRQ completions see either a live lease with kref_get_unless_zero success or no lease at all, preventing use-after-free. Retirement encodes the visibility rules by DMA type. For DMA_TO_DEVICE or DMA_BIDIRECTIONAL on non-coherent systems, CPU writes before submission require dma_sync_*_for_device before descriptor ownership transfer. For DMA_FROM_DEVICE or DMA_BIDIRECTIONAL, CPU reads after completion require dma_sync_*_for_cpu before software touches the buffer. Requeue after CPU inspection requires dma_sync_*_for_device before a new publication. The retire worker also invokes a driver or bus quiesce callback when the lease completed with timeout, reset, or failure so the device is not assumed idle merely because software stopped waiting. After quiesce, the worker performs dma_unmap_* and then invokes a new optional iommu_ops->drain_iotlb(domain, lease) callback. If the IOMMU or bus reports active ATS, PASID, PRI, or SVA without such a drain callback, dma_lease_prepare never permitted the lease in the first place and returns -EOPNOTSUPP. Only after DMA unmap and explicit drain success does the worker call the CoCo re-private path, unpin user pages or release kernel pages, remove the lease from the xarray, and wake drain waiters.

2.4 Architecture

Kernel integration is an opt-in library rather than a global DMA-API semantic change.

ASCII control flow:

process context submitter
    |
    v
 dma_lease_prepare()
    -> classify source
       * kernel pages or folios: require whole-page or whole-folio ownership
       * page_pool: require full-page object, no fragment sharing
       * user anon: pin_user_pages* + FOLL_PIN (+ FOLL_LONGTERM when declared)
       * reject slab, vmalloc, file-backed, DAX, KSM, shared-anon, hugetlb, PFNMAP
    -> normalize or split to order-0 pages when splittable
    -> coco_ops.make_shared(pfn range)  [x86 CoCo shared-memory path]
    -> dma_map_page_attrs or dma_map_sgtable
    -> optional dma_lease_publish_desc()
         shadow validate -> pack -> shared slot copy
         dma_sync_*_for_device if non-coherent descriptor memory
         dma_wmb()
         set OWN bit
         dma_wmb()
         writel() doorbell
         mmiowb() before unlock if relaxed MMIO in producer lock
    -> state = INFLIGHT

hardirq or softirq completion
    |
    v
 dma_lease_complete(lease_id, status)
    -> rcu_read_lock()
    -> xa_load(lease_id)
    -> kref_get_unless_zero()
    -> atomic cmpxchg(INFLIGHT -> COMPLETED or FAILED)
    -> queue_work(retire_wq, &lease->retire_work)

retire worker (process context only)
    |
    v
 retire_work()
    -> if timeout/reset: driver quiesce or reset callback
    -> for FROM_DEVICE or BIDIRECTIONAL CPU access on non-coherent:
         dma_sync_*_for_cpu()
    -> dma_unmap_*()
    -> iommu_ops->drain_iotlb() or reject configuration up front
    -> coco_ops.make_private(pfn range)
    -> unpin or recycle pages
    -> xa_erase(lease_id)
    -> wake_up(domain->drain_wq)
    -> kref_put()

Core data structures:
- struct dma_lease_domain { struct device *dev; spinlock_t lock; struct xarray leases; wait_queue_head_t drain_wq; struct workqueue_struct *retire_wq; unsigned long flags; refcount_t refs; const struct dma_lease_coco_ops *coco_ops; const struct dma_lease_iommu_ops *iommu_ops; }
- struct dma_lease { u64 lease_id; refcount_t refs; struct rcu_head rcu; struct work_struct retire_work; atomic_long_t state; enum dma_data_direction dir; struct sg_table sgt; struct page **pages; unsigned int nr_pages; dma_addr_t iova; size_t len; unsigned long submit_jiffies; unsigned long flags; struct dma_lease_domain *domain; void *shadow_desc; }

Concurrency and ordering rules:
- Domain xarray mutations are under spin_lock_irqsave(domain->lock).
- IRQ lookup is RCU protected; lease freeing is deferred until after synchronize_rcu.
- State transitions use cmpxchg on atomic_long_t state bits plus smp_store_release on terminal status visible to waiters.
- Descriptor publication uses dma_wmb, not a plain wmb, because the descriptor is device-visible DMA memory.
- MMIO doorbells use writel by default; if a relaxed accessor is chosen for a known platform, the profile requires explicit mmiowb or equivalent flush before releasing the producer lock.
- Early re-private is structurally prevented because only the retire worker owns DMA_UNMAPPED -> IOTLB_DRAINED -> REPRIVATE transitions.

2.5 Implementation Plan

1. Add include/linux/dma-lease.h with public enums, request structs, descriptor-schema structs, and API declarations: dma_lease_domain_create, dma_lease_domain_destroy, dma_lease_prepare, dma_lease_publish_desc, dma_lease_complete, dma_lease_fail, dma_lease_abort_sync, dma_lease_domain_drain. 2. Implement kernel/dma/dma-lease.c as the core library. This file owns the xarray of outstanding leases, the refcount and RCU lifetime model, the prepare-state classifier, timeout bookkeeping, the workqueue-driven retirement path, and tracepoints for prepare, map, publish, complete, forced-fail, unmap, drain, and re-private. 3. Add arch/x86/coco/dma-lease-x86.c implementing struct dma_lease_coco_ops for TDX and other CoCo guests using the existing x86 shared-memory transition path behind set_memory_decrypted and set_memory_encrypted style helpers. This layer rejects unsupported aliasing or hugepage cases and performs any required direct-map maintenance before returning success. 4. Extend include/linux/iommu.h with an optional lease-oriented drain hook, for example iommu_lease_drain_iotlb or an equivalent method in iommu_ops. The default implementation may be NULL. If NULL and the device or domain enables ATS, PASID, PRI, or SVA, the lease prepare path fails with -EOPNOTSUPP rather than pretending unmap is sufficient. 5. Provide a small descriptor helper in kernel/dma/dma-lease-desc.c that validates schema bounds, alignment, endianness, and field types, then copies from a CPU-private shadow descriptor into a core-owned shared slot so validation cannot be bypassed by post-check mutation. 6. Wire initial driver adopters through explicit library calls rather than implicit dma_map_ops interception. Good first consumers are drivers that already maintain per-request descriptors and have reset callbacks, such as a confidential-guest virtio path, a paravirtual block path, or a bounded NIC TX or RX queue. The invention does not require all drivers to change at once. 7. Add Documentation or rst guidance spelling out accepted source types, pinning rules, coherent versus non-coherent sync requirements by DMA direction, ATS or PASID denial behavior, and teardown expectations so the protocol is repeatable across subsystems. 8. File-level touchpoints are deliberately narrow: include/linux/dma-lease.h, kernel/dma/dma-lease.c, kernel/dma/dma-lease-desc.c, arch/x86/coco/dma-lease-x86.c, include/linux/iommu.h plus a small drivers/iommu hook implementation, and per-driver call sites. This keeps ownership clear: the DMA lease state machine lives in the DMA core helper, CoCo transitions live in arch x86, and IOMMU translation-drain support lives behind a dedicated callback rather than informal driver comments.

2.6 Validation Plan

Validation is both functional and adversarial. First, KUnit tests cover the pure state machine: rejected source types, allowed transition graph, IRQ-only completion transitions, teardown while inflight, and refcount plus RCU lifetime under late completions. Second, x86 confidential-guest tests in a TDX VM exercise the real CoCo backend with order-0 pages, splittable THP, unsplittable hugepages, page_pool pages, and anonymous user pages pinned through pin_user_pages*; these tests confirm that only accepted classes proceed in place and that rejected classes bounce. Third, DMA-API tests differentiate coherent and non-coherent devices and verify exact sync sequences for DMA_TO_DEVICE, DMA_FROM_DEVICE, and DMA_BIDIRECTIONAL, including descriptor publication ordering. Fourth, IOMMU-focused tests enable and disable ATS or PASID capable endpoints and confirm that prepare returns -EOPNOTSUPP when translation-drain support is absent and proceeds only when the drain hook reports completion. Fifth, reset and timeout fault-injection tests force completion races, map failures after successful make_shared, and driver detach during inflight DMA to confirm that pages are not returned to their allocator before unmap, drain, and re-private complete. Sixth, descriptor-schema tests inject raw kernel pointers, PFNs, raw physical addresses, misaligned fields, overlapping fields, wrong endianness encodings, and post-validation mutations to verify that the shadow-copy publication path blocks TOCTOU and metadata leakage. Seventh, performance measurements compare domain-managed in-place conversion versus bounce fallback in a TDX guest, focusing on exposure duration, descriptor latency, and queue throughput without relying on any forbidden lazy xstate or exception-driven optimization.

4. Risks & Mitigations

  • Risk: some IOMMU stacks will not initially expose a reliable translation-drain hook for ATS, PASID, PRI, or SVA. Mitigation: make strict rejection the default; the feature is enabled only on devices and domains that either disable those capabilities or implement the hook, so correctness is preserved even if coverage is narrower at first.
  • Risk: the user-page exclusivity filter may reject many real workloads because proving safe in-place conversion is intentionally conservative. Mitigation: the design includes an explicit domain-managed shared bounce path so functionality remains available while keeping the patentable core on legally convertible pages.
  • Risk: splitting THP or large folios can add latency and MM complexity. Mitigation: splitting is optional and occurs only in sleepable prepare context; unsplittable or policy-disallowed folios are bounced instead of being half-supported.
  • Risk: descriptor-schema validation could become too rigid for varied device formats. Mitigation: the schema is intentionally typed and extensible, but every new field type must still declare alignment, width, and leak properties; unsupported vendor formats can bypass descriptor publication and still use the lease for payload exposure alone.
  • Risk: domain drain during frequent resets could delay driver teardown because pages are withheld until re-private completes. Mitigation: the design favors safety over rapid free, provides forced-fail paths and wait queues so teardown remains bounded and observable, and never silently returns exposed pages to allocators.
