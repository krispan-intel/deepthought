2. Technical Invention Disclosure

2.1 Problem Statement

In memory-intensive workloads—aggressive large mmap operations, filesystem cache thrashing during large file ingest, post-fork collapse storms in JVM or Redis—khugepaged scanning and compaction can contribute 10–20% of package power through sustained CPU cycles in page migration, memory traffic from scanning PTEs across gigabytes of address space, and compaction's repeated memory reads/writes that generate DRAM row activations and precharge cycles measurable via MSR_DRAM_ENERGY_STATUS. Power-capped datacenter deployments operating near RAPL PL1 limits (common in hyperscale environments where Xeon-based multi-socket systems represent a $15B+ annual market segment) and thermal-limited edge devices running these workloads need automatic THP throttling when measured compaction power exceeds per-node quotas, but existing THP modes (always/madvise/never) and defrag settings (always/defer/defer+madvise/madvise/never) are static sysfs knobs that cannot react to runtime power attribution. Furthermore, the energy cost of compaction is entangled with baseline DRAM refresh power, background memory scrubbing, and unrelated workload memory traffic, making naive RAPL readings insufficient for policy decisions without proper isolation of compaction-attributable energy. Current ACPI HMAT-based memory tiering (Linux 6.1+ memory_tier infrastructure) provides static per-node energy attributes set at boot from firmware tables, but these attributes reflect manufacturer specifications under reference conditions, not live workload-dependent power profiles—a node whose HMAT attributes indicate low energy cost may exhibit high runtime DRAM power under memory-intensive compaction storms, a divergence that static hints cannot capture.

2.2 Prior-Art Gap

intel_rapl exposes package power via MSR_PKG_ENERGY_STATUS and DRAM power via MSR_DRAM_ENERGY_STATUS, and khugepaged exposes scan_sleep_millisecs and pages_collapsed counters, but no kernel mechanism exists that: (a) attributes package power specifically to memory management operations by correlating khugepaged CPU time accounting (task->se.sum_exec_runtime) with RAPL energy deltas and IMC uncore PMU transaction counts (specifically UNC_M_CAS_COUNT.RD event code 0x04 umask 0x03 and UNC_M_CAS_COUNT.WR event code 0x04 umask 0x0C on Sapphire Rapids/Emerald Rapids, with Granite Rapids requiring discovery-table-based remapping due to its split-die IMC topology with separate near-HBM and far-DDR5 controllers) to separate compaction-driven memory traffic from workload-driven traffic; (b) subtracts per-node baseline DRAM energy using a three-phase calibration methodology: (i) boot-time idle calibration over a 5-second quiescent window per node immediately after memory initialization but before workload admission, stored in per-node pamcg_baseline_uj; (ii) periodic re-calibration during detected idle intervals (when per-node IMC CAS transaction rate falls below 1% of peak for ≥500ms), exponentially weighted with α=0.1 toward the boot-time baseline to resist transient measurement noise; (iii) optional per-DIMM SPD-based floor from JEDEC-specified IDD6 (self-refresh current) values as a sanity lower bound—avoiding conflict with existing powercap sysfs or turbostat idle accounting since those tools report aggregate domain energy, not per-operation attributed energy; (c) detects when compaction CPU cycles and attributed memory traffic exceed per-node quotas and automatically reduces promotion aggressiveness with multi-stage backoff. Existing power governors (intel_pstate, cpufreq) target CPU frequency/voltage and do not isolate or modulate memory subsystem operations. Existing NUMA balancing (task_numa_fault, numa_migrate_preferred, node_demotion[] arrays, next_demotion_node()) uses distance-based and access-frequency heuristics with no energy dimension. The ACPI HMAT specification provides static energy-related memory attributes for NUMA tiering decisions consumed by mm/memory-tiers.c at boot, but these are manufacturer-specified constants that do not vary with runtime workload conditions; our live sampling approach continuously measures actual hardware energy counters and adapts policy on 2-second timescales, a fundamentally different mechanism from static table lookup. The energy normalization approach of dividing RAPL DRAM energy by NUMA faults plus migrations would conflate cause and effect since NUMA hint faults trigger migrations, double-counting kernel-generated events; proper normalization requires hardware-observable denominators such as IMC transaction counts that are incremented by the memory controller independently of kernel fault accounting.

2.3 Proposed Invention

Introduce a Power-Aware Memory Compaction Governor (PAMCG) that integrates RAPL energy sampling with khugepaged CPU time accounting and IMC uncore PMU counters to isolate compaction-attributable power with temporal precision. **Energy Attribution Method:** Each khugepaged scan cycle (default period aligned to scan_sleep_millisecs, typically 10 seconds) triggers a synchronized sample of: (i) MSR_PKG_ENERGY_STATUS and MSR_DRAM_ENERGY_STATUS deltas via the intel_rapl powercap interface; (ii) per-thread CPU time delta from task->se.sum_exec_runtime for all khugepaged and kcompactd threads on the node; (iii) IMC uncore PMU counters UNC_M_CAS_COUNT.RD (event 0x04, umask 0x03) and UNC_M_CAS_COUNT.WR (event 0x04, umask 0x0C) on Sapphire Rapids and Emerald Rapids—on Granite Rapids, the split-die IMC topology requires discovery-table-enumerated event codes and separate accounting for near-HBM and far-DDR5 controllers, handled by a microarchitecture abstraction layer that maps logical IMC indices to physical event selectors via cpuid family/model/stepping dispatch. The compaction-attributed energy fraction is computed as: E_compaction = E_pkg × (T_compact / T_total) + (E_dram - E_baseline) × (CAS_compact_window / CAS_total_window), where CAS_compact_window is the delta in IMC CAS read+write transactions during the khugepaged active scan phase (measured by bracketing the scan with PMU reads) and CAS_total_window is the full-interval CAS delta. E_baseline is the per-node idle DRAM power determined by the three-phase calibration: boot-time quiescent measurement, periodic re-calibration during detected idle intervals (CAS rate < 1% peak for ≥500ms, EWMA α=0.1), and JEDEC SPD IDD6 floor. **Addressing Circular Dependency in IMC Counts:** Because IMC transaction counts include all DRAM traffic—including traffic generated by PAMCG-triggered migrations and compaction—a naïve feedback loop could create positive-feedback oscillations on borderline-energy nodes. PAMCG addresses this through three mechanisms: (i) the CAS_compact_window measurement brackets only the khugepaged scan phase, not the entire interval, so migration traffic outside the scan window is excluded from the attribution numerator; (ii) a dampening coefficient (default 0.7) is applied to the compaction energy estimate when the governor is in BACKOFF state, preventing the energy attributed to residual compaction activity from triggering further escalation; (iii) the hysteresis band (high threshold 85%, low threshold 65% of per-node power quota) ensures that small perturbations in transaction mix from policy-induced migration changes do not cause state oscillations—the system must sustainably cross the threshold for the full short-window duration (2 seconds) before transitioning. **Dual-Window Temporal Model:** A short window (2-second, 8 samples at 250ms granularity) provides phase-change responsiveness; a long window (10-second, 40 samples) provides stable energy baselines. The phase-change detector computes per-page energy variance σ²(E/page) over the short window and triggers a history flush when σ² > 2 × μ_ewma(E/page) for 3 consecutive short-window samples, where μ_ewma is the exponentially weighted moving average with α=0.2. Post-flush behavior: the governor enters a CONVERGING sub-state with a minimum occupancy requirement of 4 valid samples (1 second) before re-enabling energy-based policy decisions; during CONVERGING, the governor falls back to energy-agnostic defaults (equivalent to current kernel behavior: defrag=defer+madvise, no folio order caps), preventing oscillation from insufficient data. Flush hysteresis prevents repeated flushes: after a flush event, the next flush is suppressed for at least 2 full short-window cycles (4 seconds) regardless of variance readings. **Three-State Governor:** Per-node states NORMAL, BACKOFF, and SUPPRESS with configurable thresholds (default: NORMAL→BACKOFF at 85% of per-node power quota, BACKOFF→SUPPRESS at 95%, SUPPRESS→BACKOFF at 80%, BACKOFF→NORMAL at 65%). In NORMAL: default kernel THP/compaction behavior. In BACKOFF: khugepaged scan_sleep_millisecs doubled, defrag level reduced to defer, folio order cap reduced from order-9 to order-4. In SUPPRESS: khugepaged scanning paused, compaction defrag set to never for non-madvise regions, folio order cap set to order-0 (base pages only). Per-node energy class is published via seqcount_t, chosen over RCU because: (a) the single-writer design (one pamcg kthread per node) eliminates writer-writer contention, making seqcount_t's simple retry-on-conflict model sufficient; (b) read-side cost is a single memory barrier with no preemption-disable overhead, critical for page-fault hot paths where rcu_read_lock()'s preempt_disable/enable adds measurable overhead under CONFIG_PREEMPT_FULL; (c) on 8-socket systems with 100+ concurrent faulting threads, per-cpu mirroring would require O(NR_CPUS) storage per node and cross-CPU synchronization on updates, whereas seqcount_t uses a single cacheline per node with writer updates at 250ms granularity (negligible contention). Fault-path consumers read via READ_ONCE after seqcount validation, adding zero locks to the allocation hot path. **NUMA Energy-Aware Migration Gating:** PAMCG injects per-node energy_class (NORMAL/BACKOFF/SUPPRESS) into task_numa_compare() and should_numa_migrate_memory() as an additional gating signal. When a destination node is in SUPPRESS, migration toward that node is blocked unless the source-to-destination NUMA distance exceeds a configurable locality_override_threshold (default: 30, corresponding to 2+ hop distance), which prevents the pathological case where all nodes report high energy and migration is universally blocked. Specifically, the energy-vs-locality tiebreaker operates as follows: (i) if destination energy_class is NORMAL, migration proceeds per existing AutoNUMA policy; (ii) if BACKOFF, migration proceeds only if the expected access latency improvement (estimated from NUMA distance delta × numa_balancing_scan_period_min) exceeds a minimum energy differential threshold of 5% of per-node quota; (iii) if SUPPRESS, migration is blocked unless locality_override_threshold is exceeded OR the source node is also in SUPPRESS state (preventing deadlock by allowing inter-SUPPRESS migration when no better option exists). A minimum_energy_differential parameter (default: 5% of per-node power quota) prevents migration churn between nodes with similar energy profiles. This gating is explicitly overridable: userspace can set pamcg_numa_override=1 via sysfs to disable energy gating entirely, falling back to pure AutoNUMA distance/frequency heuristics. **Integration with Linux Memory Tiering API:** PAMCG operates as a parallel energy classification that complements—not replaces—the existing memory_tier infrastructure in mm/memory-tiers.c. Rather than extending struct memory_tier (which would create coupling with ACPI HMAT initialization and CXL memory expander enumeration), PAMCG maintains a separate per-node energy_class that is consulted at the node_demotion selection point: when next_demotion_node() selects a candidate based on existing latency/bandwidth tiers, PAMCG interposes a check that vetoes demotion to SUPPRESS-state nodes and annotates BACKOFF-state nodes with an energy cost penalty that the existing tier-walk comparator can incorporate. This is implemented via a notifier callback registered with memory_tier_notifier_chain that updates demotion path weights when energy classifications change, avoiding direct modification of the memory_tier structs. For interaction with DAMON-based TPP (Tiered Page Placement) and demote-on-reclaim in kswapd, PAMCG exposes its per-node energy class via a lightweight accessor function pamcg_node_energy_class(int nid) that DAMON schemes and kswapd demotion paths can optionally query, gated by a CONFIG_PAMCG_TIER_INTEGRATION Kconfig option to allow builds without the dependency.

2.4 Architecture

PAMCG is structured as four kernel components: (1) **pamcg_sampler**: a per-node kthread that wakes every 250ms to read RAPL MSRs (MSR_PKG_ENERGY_STATUS at 0x611, MSR_DRAM_ENERGY_STATUS at 0x619) and IMC uncore PMU counters (UNC_M_CAS_COUNT.RD/WR), computes attributed energy via the CPU-time-ratio + CAS-ratio formula, maintains dual-window EWMA state, and runs the phase-change detector; (2) **pamcg_governor**: consumes sampler output, implements the three-state FSM with hysteresis, and publishes per-node energy_class via seqcount_t to a per-node pamcg_node_state structure in a statically allocated array indexed by node ID; (3) **pamcg_policy**: hooks into khugepaged (modulating scan_sleep_millisecs and pages_to_scan via exported tunables), compaction (adjusting defrag level via a new compaction_energy_override that takes priority over the sysfs defrag knob when PAMCG is active), and page fault THP allocation (reading energy_class via READ_ONCE in the fault path to cap folio order); (4) **pamcg_numa**: registers callbacks in the NUMA balancing path to gate migration decisions based on destination-node energy_class with the locality-override tiebreaker logic, and registers with memory_tier_notifier_chain to adjust demotion path weights.

Microarchitecture abstraction is handled by a pamcg_arch_ops structure with function pointers for IMC event configuration, CAS counter reading, and baseline calibration, dispatched at module init via cpuid family/model/stepping matching. Initial support covers Sapphire Rapids (ICX_IMC uncore driver, event 0x04 umask 0x03/0x0C), Emerald Rapids (same encoding, different CPUID model), and Granite Rapids (discovery-table enumeration with split-die near/far controller mapping). The abstraction allows future extension to AMD EPYC platforms via RAPL-equivalent AMD Energy Driver (amd_energy) and Data Fabric PMU counters without modifying core governor logic.

Data flow: pamcg_sampler → pamcg_governor (energy attribution + FSM) → pamcg_node_state[nid].energy_class (seqcount_t) → pamcg_policy (THP/compaction modulation) + pamcg_numa (migration gating + tier integration). All communication is lockless via seqcount_t or atomic reads; no new locks are added to any existing hot path.

4. Risks & Mitigations

  • **Risk: RAPL energy counter granularity and wraparound.** MSR_PKG_ENERGY_STATUS and MSR_DRAM_ENERGY_STATUS are 32-bit counters with platform-specific energy unit resolution (typically 15.3μJ on Sapphire Rapids). At high power (250W package), the counter wraps every ~65 seconds. **Mitigation:** pamcg_sampler reads counters every 250ms (well within wraparound period), detects wraparound via unsigned subtraction arithmetic that correctly handles single-wrap overflow, and flags double-wraparound (>65s stall) as invalid sample to be discarded rather than misinterpreted.
  • **Risk: IMC uncore PMU counter availability and contention.** Uncore PMU counters are a shared resource; other profiling tools (perf, VTune) may compete for counter slots. **Mitigation:** PAMCG uses perf_event_create_kernel_counter() with exclusive=0, allowing co-existence with userspace profiling. If counter allocation fails (all slots occupied), PAMCG falls back to CPU-time-ratio-only attribution (omitting the CAS-ratio term), which provides degraded but functional power estimation. Sysfs status file reports active attribution mode.
  • **Risk: All-nodes-SUPPRESS pathological case causing NUMA migration deadlock.** If a system-wide memory pressure event causes all NUMA nodes to enter SUPPRESS state simultaneously, migration gating could trap pages on remote nodes indefinitely. **Mitigation:** The locality-override tiebreaker explicitly permits inter-SUPPRESS migration when source and destination are both in SUPPRESS state, allowing AutoNUMA to continue distance-based optimization. Additionally, a system-wide pamcg_emergency_release sysfs trigger allows operators to force all nodes to NORMAL state, and the governor includes an automatic all-SUPPRESS timeout (default: 30 seconds) that transitions the highest-capacity node to BACKOFF to break the deadlock.
  • **Risk: Phase-change detector false flushes causing policy instability.** If the variance threshold is too sensitive, repeated flushes prevent stable energy baselines from forming, causing the governor to oscillate between energy-aware and energy-agnostic modes. **Mitigation:** Post-flush hysteresis suppresses subsequent flushes for 2 full short-window cycles (4 seconds). The CONVERGING sub-state requires 4 valid samples before re-enabling policy. The threshold formula (σ² > 2×μ_ewma for 3 consecutive samples) is deliberately conservative—requiring 3 consecutive exceedances at 250ms granularity means 750ms of sustained high variance before flush, filtering out transient spikes. All parameters are tunable via sysfs for workload-specific adjustment.
  • **Risk: Circular dependency between policy actions and IMC measurements.** PAMCG-triggered throttling of compaction changes the CAS transaction mix, which feeds back into the energy attribution formula, potentially creating positive-feedback loops on borderline-energy nodes. **Mitigation:** Three-layer dampening: (i) CAS_compact_window brackets only the scan phase, excluding migration traffic; (ii) 0.7 dampening coefficient in BACKOFF state reduces attributed energy from residual compaction; (iii) 20-percentage-point hysteresis band (85% up, 65% down) ensures small perturbations do not trigger state transitions. Validation plan includes explicit oscillation detection testing.
  • **Risk: Microarchitecture divergence across Intel platforms.** Sapphire Rapids, Emerald Rapids, and Granite Rapids have different IMC topologies (monolithic vs. split-die), different uncore PMU event encodings (fixed vs. discovery-table), and different RAPL domain configurations. **Mitigation:** pamcg_arch_ops abstraction layer with per-platform function pointers dispatched at module init via CPUID. Granite Rapids split-die support uses the uncore discovery table (already parsed by the intel-uncore-frequency driver) to enumerate IMC instances. Unsupported platforms receive a graceful -ENODEV at module load rather than silent malfunction.
  • **Risk: Competitive positioning against AMD EPYC.** AMD's NUMA implementation with Infinity Fabric has different power and topology characteristics. **Mitigation:** The pamcg_arch_ops abstraction explicitly supports future AMD integration via the amd_energy driver (RAPL-equivalent) and Data Fabric PMU counters. The governor logic, policy hooks, and NUMA gating are architecture-independent; only the sampler layer requires platform-specific adaptation. This positions PAMCG as a vendor-neutral memory power optimization framework, strengthening adoption potential across the $25B+ combined Xeon/EPYC datacenter processor market.
