2. Technical Invention Disclosure

2.1 Problem Statement

Transparent Huge Page collapse and adjacent PMD-scale memory-management scans spend significant CPU time revisiting the same 512-entry windows under the page-table lock, executing branch-heavy per-entry logic even when the result is merely that the window is sparse, cold, or otherwise a poor collapse target. On large Xeon systems this raises lock hold time variance, burns worker cycles, and degrades promotion efficiency for OLTP, JVM, analytics, and in-memory database workloads whose address spaces contain many repeatedly scanned but weak candidates. The technical challenge is that Linux mm correctness leaves very little room for speculative acceleration: the worker cannot sleep or reschedule while using kernel vector state, cannot dereference live page-table state off-lock, cannot rely on unstable or arch-private PTE encodings, and cannot allow accelerator output to authorize collapse or permanently suppress candidates. A useful invention therefore must preserve the exact baseline correctness path while extracting only those snapshot facts that are stable enough to rank work off-lock. The invention targets precisely that gap by making the fast path advisory-only, restricting locked capture to one PMD window per snapshot, encoding only helper-defined predicates plus a mandatory UNKNOWN bucket, and ensuring that any stale or imperfect advisory result can at most delay work for a bounded interval before the ordinary locked path reconsiders the candidate.

2.2 Prior-Art Gap

Current mm/khugepaged.c already performs a bounded HPAGE_PMD_NR scan under PTL and computes eligibility and skip conditions inline, so a bare claim to scanning 512 PTEs faster would be vulnerable to obviousness and overlap. The missing prior-art element is a formalized descriptor protocol: existing code does not define a reusable, versioned, immutable PMD snapshot object that is produced under PTL, published safely to off-lock consumers, and interpreted identically by multiple reduction backends. It also does not define a strict helper-only extraction set with a mandatory UNKNOWN taxonomy that prevents off-lock reasoning about swap entries, migration entries, device-private or devmap entries, uffd-wp markers, pte_special mappings, PROT_NONE encodings, or arch-private bit patterns. Nor does current practice provide a backend-neutral contract under which scalar, AVX-512, and AMX reducers are interchangeable implementations of the same advisory score, or a bounded starvation policy proving that reordered candidates cannot be suppressed indefinitely. Finally, known AMX material teaches dense matrix accumulation but not a kernel-mm-safe envelope for applying tile instructions inside a khugepaged worker: there is no prior description of how AMX feature gating, XCR0 and IA32_XFD checks, kernel_fpu_begin/end bracketing, no-reschedule slices, CPU migration control, and fallback behavior combine with Linux THP scan ordering while leaving authoritative collapse checks untouched.

2.3 Proposed Invention

The invention inserts an optional advisory ranking stage ahead of the expensive parts of khugepaged while leaving the authoritative collapse path unchanged. The stage runs only for PMDs already admitted by existing VMA and PMD eligibility gates, so it never manufactures new candidates and never expands the search space. For each candidate, khugepaged acquires the existing PTE lock with pte_offset_map_lock() and inspects at most HPAGE_PMD_NR entries exactly once. Under the PTL it performs only constant-work helper tests and bit stores; it does not fault in pages, follow struct page pointers, walk reverse maps, resolve migration, swap in memory, invalidate TLBs, invoke MMU notifier slow paths, or retain live PTE pointers after unlock. Each PTE is normalized into one of three categories: HOLE if and only if pte_none() is true; NORMAL_PRESENT if pte_present() is true and none of the mandatory UNKNOWN conditions apply; or UNKNOWN otherwise. The fixed descriptor contains five canonical bitplanes for HOLE, NORMAL_PRESENT, WRITABLE, YOUNG, and DIRTY, with WRITABLE, YOUNG, and DIRTY only set when NORMAL_PRESENT is set. Any entry that is swap-like, migration-like, device-private, devmap, uffd-wp, pte_special, PROT_NONE, helper-unavailable, or otherwise arch-private contributes no attribute bits and is recorded only through UNKNOWN reason counters. Raw pte_t values are not exported off-lock. Tier-0 early abort is intentionally limited to HOLE detection: if any HOLE bit is set, the descriptor receives a fixed low advisory score and HOLE_PRESENT explanation because a fully populated collapse cannot be authorized from that snapshot; this is safe even under concurrent faults because the result is advisory only and bounded anti-starvation guarantees later reconsideration. No other PTE state may trigger early abort because its semantics are architecture- or context-sensitive. After capture, the producer publishes the descriptor as an immutable allocation via release semantics; consumers acquire the pointer, read byte-stable contents, and score the descriptor off-lock with a scalar reducer, an AVX-512 reducer, or, where explicitly permitted by kernel xstate policy, an AMX INT8 reducer. The reducer computes only an advisory priority and explanation class, for example HOLE_PRESENT, UNKNOWN_HEAVY, COLD_CLEAN, DENSE_ACTIVE, or AGE_FORCED. Queue ordering may use the score to defer weak candidates, but every candidate has a bounded defer count and age-based override so none can starve. When a candidate is finally selected for actual collapse, khugepaged re-enters the existing locked scalar revalidation and collapse code path; descriptor output neither authorizes collapse nor suppresses any mandatory baseline checks.

2.5 Implementation Plan

Phase 1 adds the normalized descriptor type, helper-only extraction routine, and immutable publication path, with the scalar reducer as the sole initial backend and a debug mode that recomputes counts directly from the bitplanes to validate ABI consistency. Phase 2 wires the capture hook into khugepaged_scan_pmd(), stores descriptors per discovered candidate, and adds explanation enums and counters without changing the collapse decision path. Phase 3 introduces bounded queue ordering in khugepaged_scan_mm_slot(), including defer_count tracking, first-seen timestamps, and the age-forced FIFO override so the scheduler effect is observable and bounded. Phase 4 adds the AVX-512 backend behind a feature gate and continuously cross-checks it against the scalar reducer on the same immutable descriptor bytes during soak testing. Phase 5 adds the AMX backend as a non-essential x86 embodiment only if the target kernel exposes an explicit policy allowing in-kernel XTILE usage; that phase includes the capability hook for CPUID, XCR0, and IA32_XFD checks, the kernel_fpu_begin/end bracketing, tile configuration caching, 16-descriptor micro-batching, tilerelease on exit, and automatic fallback to AVX-512 or scalar on any precondition failure. Phase 6 measures success using lock-hold histograms, khugepaged CPU time, collapse attempts per successful promotion, descriptor staleness rate, and latency impact under sparse, dense-hot, and adversarial-mixed workloads. A deployment rule keeps the invention behavior-safe: if the reducer queue is disabled, if vector state is unavailable, or if debug cross-checking reports any semantic mismatch, khugepaged reverts to the baseline scalar ordering while preserving the rest of the MM subsystem unchanged.
