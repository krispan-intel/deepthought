3. Technical Invention Disclosure

3.1 Problem Statement

Linux MGLRU improves reclaim quality, but the hot path that rotates a single folio between lrugen lists inside one lruvec is still serialized by lru_lock even when the caller already holds a stable folio reference and the actual mutation is only one unlink, one relink, and a small accounting update. On high-core-count x86 NUMA servers, many CPUs can concurrently discover that individual folios should be promoted, demoted, or retiered, yet each one-folio move still transfers the same lru_lock cacheline and nearby lrugen metadata between sockets. That lock transfer amplifies tail latency and reduces reclaim scalability. The challenge is that mm cannot simply perform lockless list moves: a staged move can become invalid if the folio is isolated for reclaim, compaction, or migration; if memcg ownership changes; if nid changes; if generation or tier membership changes; if memcg offline reparents the folio to a different lruvec; if memory hotplug rebuilds lruvec relationships; or if security policy disables TSX entirely. The problem to solve is therefore narrowly defined: reduce lru_lock hold time for same-lruvec, single-folio MGLRU rotations without weakening folio lifetime safety, memcg and node identity correctness, exact accounting correctness, forward progress under RTM aborts, or correctness when TSX is disabled, masked, or permanently force-aborting.

3.2 Prior-Art Gap

Known techniques leave a specific gap. qspinlock optimistic spinning and historical lock-elision work are generic lock-acquisition mechanisms; they do not define a memory-management-specific validity contract for a staged MGLRU move, do not require a caller-held folio reference for safe lockless sampling, and do not re-derive memcg, nid, lruvec, generation, tier, and exact accounting at the commit consistency point. Generic seqcount and seqlock patterns can cheaply invalidate stale observations, but seqcount plus conventional lock acquisition still leaves the full lru_lock handoff, hold time, and cacheline bouncing intact because the actual list mutation and accounting still happen only after the lock is acquired. Existing MGLRU lockless prechecks and heuristics are advisory only and stop short of committing the list move itself. Existing optimistic memory-reclamation work also fails to enumerate the exhaustive Linux-specific invalidation set required here: list membership changes, generation advancement, tier changes, isolate and putback, eviction and refault reinsertion, memcg migration or offline reparenting, node migration or lruvec rebuild, and order-eligibility transitions. The gap is a claim-ready mechanism that combines an exhaustive per-lruvec invalidation contract, a pinned-folio advisory capsule, a full-lockword qspinlock subscription, commit-point re-derivation of identity and accounting, and an immediate fail-closed fallback to the existing lru_lock path.

3.3 Proposed Invention

1. Scope: Only order-0, same-lruvec, single-folio MGLRU rotations that the baseline code would already execute under lru_lock are eligible. Cross-lruvec moves are never committed transactionally. 2. Folio lifetime rule: The helper may be entered only when the caller already holds a live folio reference obtained outside the helper, such as from a page-cache, rmap, page-table, or reclaim walk. The helper never speculatively acquires a reference to an otherwise unpinned LRU folio. 3. Advisory-only capsule: Under RCU read-side protection and a read-seqcount acquire of a per-lruvec mutation sequence, the helper samples only hints: sampled memcg, sampled nid, sampled lruvec pointer, sampled generation, sampled tier, sampled type, sampled zone, sampled source and destination list identities, and sampled sequence value. These hints are used only to decide whether an RTM attempt is worthwhile. 4. Correctness-critical state: Inside RTM or, on fallback, under lru_lock, the helper recomputes from current memory the folio's current memcg, nid, lruvec, generation, tier, type, order-0 eligibility, LRU eligibility, isolate status according to the same predicate the baseline path already trusts, source list, destination list, and exact page-count delta. No delta or destination chosen from the capsule is ever applied directly. 5. Exhaustive mutation-sequence contract: Every MGLRU writer that can invalidate a staged same-lruvec move wraps its mutation with write-seqcount-begin and write-seqcount-end while holding lru_lock. Mandatory sequence coverage includes: list add, delete, or move into, out of, or between lrugen lists; generation advancement, promotion, demotion, or aging; tier changes driven by referenced or protection logic; isolate and putback for reclaim, compaction, migration, or activation and deactivation; eviction or refault reinsertion; memcg migration, memcg offline reparenting, or any ownership move between lruvecs; node migration, memory hotplug or offline, or lruvec rebuild; and any change in folio order or compound eligibility that would remove the operation from the order-0 scope. Cross-lruvec transitions bump both source and destination sequences. 6. RTM commit lane: With preemption and local interrupts disabled, xbegin starts a transaction. The helper transactionally reads the full qspinlock word for lru_lock and proceeds only if the entire word is zero, thereby subscribing to locked, pending, and queued state on the actual lockword used by lock takers. It then re-derives live folio and lruvec identity, verifies that the sampled sequence is still the same even value, verifies same-lruvec eligibility, recomputes source and destination lists and exact accounting, performs the list unlink and relink, updates the same per-generation per-type per-zone counters that the baseline helper would update, and xend commits. Any abort, mismatch, unsupported policy, or revalidation failure triggers immediate fallback. 7. Policy and observability: The fast path is default-off behind a boot-time static key. PMU data may be used to measure benefit, but PMU availability never affects correctness or eligibility. Adaptive runtime backoff uses only local RTM abort status and administrator policy, not PMU interrupts or PMU virtualization support.

Boot and policy gate
  CPUID reports RTM
  AND RTM is not disabled by microcode or kernel TSX policy
  AND system TAA policy permits TSX use
  AND no disallowing hypervisor or lockdown policy is active
      -> enable global static branch for mglru RTM path
  else
      -> always use baseline locked path

Per-CPU fast-path gate
  CPU not in cooldown
  AND abort budget is acceptable
  AND no nested RTM attempt
  AND caller already holds folio reference
      -> optimistic sample lane
  else
      -> baseline locked path

Optimistic sample lane (RCU read-side, no mutation)
  seq0 = acquire read of lruvec mutation sequence
  sample advisory hints: memcg, nid, lruvec hint, gen, tier, type, zone, src and dst hints
  retry check on seq; if unstable, fall back

RTM commit lane (preemption disabled, local interrupts disabled)
  xbegin
    lockword = full READ_ONCE of lru_lock qspinlock word
    require lockword == 0
    re-derive current memcg and nid from folio
    re-derive current lruvec from memcg and nid
    require current lruvec matches sampled lruvec
    require mutation sequence still matches sampled even value
    require folio is still pinned, order-0, LRU eligible, and not isolated according to baseline predicate
    recompute current generation, tier, type, source list, destination list, and exact delta
    unlink folio from current list
    relink folio to destination list
    update per-generation per-type per-zone page counters and any coupled metadata already changed by baseline helper
  xend
  on abort -> record abort code, maybe enter cooldown, then fallback

Fallback locked lane
  spin_lock_irqsave on lru_lock
  re-derive all identity and accounting from live state
  invoke existing authoritative MGLRU rotation helper
  spin_unlock_irqrestore

Memory touched inside RTM is intentionally fixed and non-faulting: the qspinlock word, folio flags relevant to LRU and generation state, folio list_head links, source and destination lrugen list heads, per-generation counters, and the mutation sequence. No allocation, no user access, no sleeping lock, no reclaim recursion, and no indirect callback is allowed. Interrupts and NMIs are treated as abort causes only; they do not create correctness dependence.

1. Add a global static key so the compiled fast path is never reached unless boot-time policy explicitly enables it. 3. Add debug assertions so a mutation helper cannot update lrugen state without both lru_lock held and sequence coverage active. 4. Refactor the existing locked one-folio rotation helper so that all correctness-critical recomputation lives in one shared path usable by both the RTM lane and the fallback lane. 5. Implement the advisory capsule helper under RCU plus read-seqcount acquire semantics. Explicitly document in code that capsule fields are hints only and may not be used to derive accounting or destination without commit-time recomputation. 6. Implement the x86 RTM helper as a tight inline region that disables preemption and local interrupts, performs xbegin, subscribes to the full qspinlock word by requiring an all-zero state, revalidates identity and eligibility, applies the list and counter mutations, and xend commits. The helper must not call code that sleeps, faults, allocates, or enables preemption. 7. Implement bounded retry and disable heuristics: zero retries for explicit, capacity, debug, interrupt, nested, or unsupported aborts; at most one retry for pure conflict aborts; CPU-local cooldown when abort rate exceeds a threshold over a time window; optional global cooldown driven by aggregated software counters only. PMU presence is not required. 8. Add observability that is strictly non-authoritative: counters or tracepoints for attempts, commits, abort reasons, lock fallbacks, and cooldown entries, plus perf recipes using RTM retired events and lock or cache contention events for benchmarking. 9. Validate staged rollout in this order: exhaustive sequence-contract coverage and fallback refactor first, RTM commit lane second, cooldown heuristics third, default-off fleet trial last.

5. Risks & Mitigations

  • TSX and TAA security risk: keep the feature default-off and enable it only where existing kernel and platform policy already permits TSX use; automatically force the path off under lockdown, disallowing TAA policy, or uncertain hypervisor exposure.
  • Abort storms from interrupts, conflicts, or capacity pressure: allow at most one retry for conflict aborts, no retry for interrupt, explicit, nested, capacity, debug, or unsupported aborts, and use CPU-local cooldown so reclaim immediately converges to the locked path.
  • Missed mutation-sequence coverage: enumerate the exhaustive writer contract in code and test plans, couple the sequence to lru_lock with debug assertions, and treat any helper that mutates lrugen state without sequence coverage as a correctness bug.
  • qspinlock implementation drift or subscription weakness: subscribe to the full lockword rather than only a locked bit, validate the layout on supported x86 kernels, and keep the feature disabled if the underlying lock representation or acquisition semantics are not known to be compatible.
  • Folio identity drift from migration, isolate, memcg reparenting, node movement, or hotplug: require a caller-held folio reference for lifetime, re-derive memcg, nid, lruvec, generation, and list identity at commit, and fail closed to the locked path on any mismatch.
  • Accounting divergence or double-counting: compute deltas only at the commit point, update only the same list links and per-generation counters that the baseline helper updates, and never replay sampled or deferred deltas after an abort.
  • PMU unavailability in virtualized or restricted fleets: use PMU events only for measurement and tuning visibility, never for correctness gating, so the feature behaves safely and predictably even when PMU access is masked.
